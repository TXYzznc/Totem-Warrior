# Scripts — C# 脚本根目录

> 所有 C# 脚本的根目录，按功能分区。

## 目录

| 目录 | 说明 | 文件数 |
|---|---|---|
| `Core/` | 框架核心（项目通用，不可删） | 7 |
| `Modules/` | 框架可选通用模块（不含项目语义） | 2 |
| `Utils/` | 独立于业务的通用工具类 | 7 |
| `Templates/` | AI 开发时的参考模板（.cs.txt，不编译） | 2 |
| `ExternalAPI/` | 暴露给外部工具调用的 API 端点 | — |
| `Events/` | 跨模块的事件类型定义 | — |

## 使用规则

- 所有 C# 脚本必须放在上述某个子目录中，不要散落在根目录
- 框架通用模块放在 `Modules/` 下，每个模块一个文件夹，含自己的 `Events/` 子目录
- 项目业务模块不要放进框架目录，应由具体项目自行组织
- 跨模块共享的类型放在对应的顶层目录（Events、Utils）
- 模板文件用 `.cs.txt` 后缀，避免被 Unity 编译
