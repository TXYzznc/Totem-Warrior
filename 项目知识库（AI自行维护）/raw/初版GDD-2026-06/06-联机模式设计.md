# 06 联机模式设计

## 一、核心需求

来自用户的明确要求：
- ✅ 支持邀请好友联机**合作**
- ✅ 支持邀请好友联机**对抗**
- ✅ 同时支持 50 人标准大逃杀

需要用一套模式架构满足以上三个不同的社交场景。

---

## 二、推荐设计：三层模式架构

### Layer 1：标准大逃杀模式（核心循环）

| 维度 | 设计 |
|------|------|
| 模式名 | BR 标准模式 |
| 人数 | 50 人（含 AI 玩家） |
| 队伍 | Solo（主推），可选 Duo / 4 人小队 |
| 好友功能 | 邀请好友自动同队进局，**队内合作，对外厮杀** |
| 适用场景 | 正式游玩、排位、日常 BR |

**这是同时满足"好友合作 + 全局 PVP"的基础形态**——朋友和你一起进局，你们之间合作，但要对抗其他 48 人 / 队。

---

### Layer 2：纯合作 PVE 副本（推荐先做）

| 维度 | 设计 |
|------|------|
| 模式名 | 废墟探索 / 副本挑战 |
| 人数 | 2-4 人组队 |
| 队伍 | 仅好友 / 匹配的队友 |
| 玩法 | 主题废墟副本（AI 据点突袭、外星巢穴探索、病毒源调查） |
| 缩圈 | 无 |
| PVP | 无 |
| 奖励 | 局外解锁素材（角色、外观、画笔等） |

**为什么先做**：
- 联机架构压力小（4 人 << 50 人）
- 适合 MVP 阶段验证联机技术栈
- 提供局外解锁内容的稳定获取渠道
- 新玩家温和入门、朋友休闲娱乐场景

---

### Layer 3：自定义私房 ⭐

| 维度 | 设计 |
|------|------|
| 模式名 | 私房 / 自定义房间 |
| 人数 | 2-50（可调） |
| 队伍 | 房主自由配置（4v4 / 2v2v2v2 / 全员混战 / 全员合作） |
| 规则 | 可微调（圈速、补给丰度、是否开纹身限制） |
| 计入排位 | 否（仅娱乐） |
| 适用场景 | 朋友定制玩法、活动、自办比赛 |

**这是关键设计点**——一个系统同时覆盖**合作 + 对抗 + 混合**所有社交需求：

| 朋友想要 | 私房怎么开 |
|---------|------------|
| 一起打 AI | "4 人组队 vs AI" 模式 |
| 一起 PVP 训练 | "2v2 对战" 或 "全员混战" |
| 大型自办比赛 | 50 人，自定义规则 |
| 偷懒练习 build | "4 人合作 + 高补给" |

> 私房模式是同类游戏（PUBG、Apex）成熟功能，本作的实现可以参考成熟方案。

## 四、联机架构推进顺序（Solo 开发友好）

### 第 1 阶段：单机原型
**目标**：验证纹身系统、战斗手感、肉鸽循环
**联机**：暂无
**用时估计**：1-2 个月

### 第 2 阶段：2-4 人合作 PVE 副本
**目标**：最简联机架构跑通
**技术挑战**：状态同步、敌人 AI 权威性、断线重连
**联机方案候选**：Mirror、FishNet、Unity Netcode
**用时估计**：2-3 个月

### 第 3 阶段：私房系统 + 小规模 PVP（4-8 人）
**目标**：加入玩家间战斗、私房创建逻辑
**技术挑战**：PVP 同步精度、防作弊（基础）、延迟补偿
**用时估计**：2-3 个月

### 第 4 阶段：扩展到 50 人 BR
**目标**：完整大逃杀体验
**技术挑战**：专用服务器架构、人机 AI 系统、区域同步优化
**联机方案候选**：Photon Fusion 2、Mirror + 自建专服
**用时估计**：3-6 个月（最难）

## 五、技术架构推荐

### 联机框架候选对比

| 框架 | 类型 | 成本 | Solo 开发友好度 | 备注 |
|------|------|------|---------------|------|
| **Mirror** | 开源 P2P / 客户端权威 | 免费 | ⭐⭐⭐⭐⭐ | 社区活跃，资料多，适合小规模 |
| **FishNet** | 开源 P2P / 服务器权威 | 免费 | ⭐⭐⭐⭐ | 现代化设计，性能好 |
| **Photon Fusion 2** | 商业云服务 | 按用量收费 | ⭐⭐⭐⭐ | BR 场景验证过，但有费用 |
| **Unity Netcode** | 官方 | 免费 | ⭐⭐⭐ | 官方支持，但生态相对新 |

### 推荐路径

```
第 2 阶段：Mirror 或 FishNet（小规模合作）
       ↓
第 3 阶段：同上扩展到小型 PVP
       ↓
第 4 阶段：评估是否切换到 Photon Fusion 或自建专服
```

## 六、AI 玩家（人机）设计

### 角色定位
- 50 人局，根据实际玩家数动态填充
- 玩家基数小（早期版本）时人机占大头
- 玩家基数大（后期）人机比例自然下降

### 行为目标
人机应**模拟真实玩家行为**而非死板巡逻：
- ✅ 会捡宝箱、买商人物品、刻纹身
- ✅ 会躲圈、找掩体、规划移动
- ✅ 会主动 PVE、避开太强的玩家
- ✅ 会"演技"：失误、犹豫、有性格

### 实现复杂度
- AI 行为是**第 4 阶段的重点工作之一**
- 可借鉴行为树 / GOAP / 简单状态机
- 关键：让 AI 看起来"有意图"而非脚本

## 七、社交功能配套

### 必备
- 好友列表
- 邀请进局
- 公会 / 战队（可选）
- 局后回放 / 击杀镜头（展示纹身设计的关键）

### 锦上添花
- 大厅画廊（展示自己 / 别人的纹身）
- 个人成就墙
- 排行榜（按角色 / 总击杀 / 死亡宝箱拾取等）

## 八、风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| 50 人联机的同步压力 | 🔴 高 | 区域同步、优先级 LOD |
| 防作弊 | 🟡 中 | 服务器权威架构、关键判定服务器端 |
| 断线重连 | 🟡 中 | 局内有重连窗口（30 秒？） |
| 跨平台联机延迟差异 | 🟡 中 | PC 和移动端分匹配池 |
| 早期玩家少导致体验差 | 🟢 低 | 用 AI 人机填充保证局完整 |
