## 一、玩家体验目标

### 一句话

2 个主动技能槽是玩家手中的"大招按钮"——高爆发、有节奏、可预谋；与纹身被动的"自动触发"形成节奏反差，让战斗不只是贴脸输出。

### 核心情感对照

| 纹身被动（自动触发） | 主动技能（玩家决策） |
|---|---|
| 攻击/闪避/受击时自动生效 | 玩家在正确时机主动按下 |
| 持续"底力输出" | 爆发窗口、高风险高收益 |
| 体感：build 在"跑" | 体感：玩家在"出牌" |

**爽点**：冷却刚好转好、敌人密集时按下 Q，四面火球散射，所有小怪同时着火——这一帧是玩家"自己造成的"高光时刻，与纹身被动无声触发的"系统奖励"完全不同。

**MDA 三层**：
- M = 2 槽位（Q/E）× 冷却 / 充能机制 × 技能帧表 × SkillCastEvent 联动
- D = 玩家在"等冷却 / 等时机"的张力中决策出招节奏；2 槽带来更聚焦的构筑选择
- A = **Submission（节奏掌控）+ Expression（build 表达）+ Sensation（爆发感官）**

## 二、核心机制

### 2.1 2 槽位系统（Q / E）

v2.1 技能槽从 3 缩减为 2，对应键位 Q / E。R 键位不再绑定技能。

| 槽位 | 键位（PC） | 键位（手柄） | 槽位语义 |
|---|---|---|---|
| Slot 1（Q） | `Q` | 肩键 LB | 灵活技（中低冷却，轻量至中量效果）|
| Slot 2（E） | `E` | 肩键 RB | 强力技（中高冷却或充能，核心爆发）|

**设计意图**：2 槽减少认知负担，聚焦每一次主动决策；单局 10-15min 内 2 个技能已能产生完整的节奏感知，不需要第 3 槽位。槽位语义为**建议分类**，不由系统强制——玩家可自由分配任意稀有度技能到任意槽位。

**MaxSlot = 2**（MVP 不扩展）。空槽不可用，图标呈灰色锁定状态，提示文字引导玩家从商人 / 宝箱 / 事件 / Boss 获取技能。

**初始状态**：每局开始时 0 个技能，2 槽均为空。技能从局内获取（§七 获取来源）。

### 2.2 冷却与充能模型

每个技能选择以下其中一种激活模式（`ChargeModel` 字段）：

#### 模式 A：纯冷却（Cooldown）

```
按下 → 技能生效 → 进入冷却倒计时（Cooldown 秒）→ 可再次使用
```

适用于快速技和中等爆发技。冷却期间图标显示顺时针减少的冷却环，剩余时间数字显示于图标中央。

#### 模式 B：充能（Charge）

```
充能层数 = MaxCharges（初始满充）
按下一次 → 消耗 1 层 → 触发 ChargeRegenTime 后自动回复 1 层
```

适用于连续释放但需要间隔的技能（如快速投掷手雷）。图标旁显示当前充能层数（如 `2/3`）。

#### 模式 C：蓄力释放（HeldCast）

```
长按 → 蓄力进度 0→100%（HoldDuration 秒）→ 松开释放
过载窗口（OverchargeWindow 秒）内释放 = 强化版本（DamageMul × 1.3）
超过过载窗口强制释放（普通版本）
```

沿用 [02-战斗手感 §2.2](./02-战斗手感.md) 蓄力普攻的视觉/音效反馈逻辑，共用边缘光晕 + "叮"提示音。

### 2.3 技能帧表规范

技能帧表**独立于普攻帧表**，三段结构相同：

```
Startup（预备）→ Active（激活，hitbox 生效 / 效果施加）→ Recovery（恢复）
```

**与普攻的关键差异**：

| 维度 | 普攻 | 技能 |
|---|---|---|
| 可被闪避取消 | Startup / Recovery 均可 | 默认**不可**取消（`CancelableByDodge = false`）；标记则可取消 |
| Hitstop | 命中时生效 | 命中时同样触发，由 CombatModule 统一管理 |
| 输入缓冲 | 3 帧 | 2 帧（来自 [02 §2.1](./02-战斗手感.md) 表）|
| 攻速加成影响 | 被 AttackSpeedBonus 压缩 | **不受攻速影响**（技能帧表固定，保持释放节奏稳定）|

**技能默认帧基线**（供 `SkillConfig.json` 参考；实际值按具体技能配置）：

| 技能类型 | Startup (帧) | Active (帧) | Recovery (帧) | 总时长 |
|---|---|---|---|---|
| 快速投射技 | 5 | 4 | 10 | 19 帧 ≈ 317ms |
| 范围爆发技 | 8 | 6 | 16 | 30 帧 ≈ 500ms |
| 强力技（高冷却 / 史诗）| 12 | 8 | 20 | 40 帧 ≈ 667ms |

### 2.4 SkillCastEvent 与左臂联动

技能进入 Active 帧第 1 帧时，CombatModule 发布：

```csharp
SkillCastEvent { Caster: actor, SlotIndex: slot, SkillId: id, AimTarget: target }
```

`SlotIndex` 现取值范围为 `0`（Q 槽）和 `1`（E 槽）。

TattooModule 订阅此事件，将左臂纹身触发：

```
左臂 OnSkillCast → 以 [左臂图案] 方式释放 [左臂颜色] 效果
effectMagnitude = scaleStat(SkillIntensity) × scaleFactor(0.6) × colorMul × patternMul × synergyMul
```

其中 `scaleStat(SkillIntensity)` = 本次技能的 `DamageMul × BaseDamage`，即**技能越强，左臂纹身触发越强**，形成正反馈。

> **耦合边界**：CombatModule 只发事件，不感知纹身逻辑；TattooModule 只订阅 SkillCastEvent，不关心技能是哪个槽位的具体实现。

### 2.5 技能稀有度

| 稀有度 | 图标边框色 | Cooldown 范围 | DamageMul 范围 | 获取方式 |
|---|---|---|---|---|
| 普通（Common） | 灰色 | 7–12s | 1.0–1.5× | 普通宝箱 / 商人常驻 |
| 稀有（Rare） | 蓝色 | 12–18s | 1.5–2.5× | 精英宝箱 / 稀有商人 |
| 史诗（Epic） | 紫色 | 18–30s | 2.5–4.0× | 特殊事件 / Boss 掉落 |

> v2.1 冷却范围上限小幅下调（原 20/35s → 18/30s），适配单局 10-15min 使感知频率在每档有 3-5 次使用机会。

## 三、与其它系统的耦合

| 系统 | 耦合方式 |
|---|---|
| **[01-纹身构筑系统](./01-纹身构筑系统.md)** | 左臂部位的触发事件 = `SkillCastEvent`；左臂 `scaleFactor = 0.6`、`scaleStat = SkillIntensity`；元素染色：技能本身可携带元素属性（`SkillElement`），左臂纹身颜色可**覆盖**技能元素，产生"纹身改色"效果（见 §四 SkillConfig `ElementOverrideByTattoo` 字段）|
| **[02-战斗手感](./02-战斗手感.md)** | 技能释放通过 `InputModule.ShouldUseSkill(slot)` 轮询，`slot` 取值 0（Q）/ 1（E）；技能帧表使用三段式；技能释放期间默认不可被闪避取消；Hitstop 由 CombatModule 统一触发 |
| **[09-纹身师与商人NPC](./09-纹身师与商人NPC.md)** | 商人 NPC 在局内售卖技能（颜料 + 武器 + **技能** + 消耗品），每局商人随机备货 1-2 件技能，稀有度与当前局时挂钩；技能购买走商人交互 UI，确认后直接弹出"装备到槽位"面板 |
| **[11-怪物与 Boss](./11-怪物与Boss.md)** | Boss 和精英怪使用相同 `SkillConfig.json` 条目，AI 侧释放决策由 EnemyModule 按行为相位控制；普通怪无技能 |
| **[12-数值平衡与曲线](./12-数值平衡与曲线.md)** | 技能 `BaseDamage` 范围引用武器等级 BaseDamage 基准（白色 10–20 / 金色 40–60）；左臂 `scaleFactor = 0.6` 来自 `TattooPartConfig.ScaleFactor`；单次技能不应一击必杀普通怪（`effectMagnitude / enemyHP(t)` 维持在 0.3–0.8） |
| **[16-BotControllerModule](../modules/16-BotControllerModule.md)** | Smart Bot 按冷却状态 + 局势模型主动使用 Q/E 两槽技能；Light Bot 不使用技能 |

### 4.1 MaxSlot 字段说明

`SkillModule` 硬编码 `MaxSlot = 2`（不在 SkillConfig.json 中存储，不随行/列变化）。若后续需扩展槽位，修改 SkillModule 常量并同步 HUD 布局，不需改动本表结构。

### 4.2 DamageMul 平衡校验（单局 10-15min 基准）

基于 [12-数值平衡 §2.4](./12-数值平衡与曲线.md)，技能单次有效伤害估算（以 t=12min、武器 BaseDamage=15 为基准）：

```
技能本体（不含纹身）：
  普通技 DamageMul=1.2 → 15 × 1.2 = 18 点
  稀有技 DamageMul=2.0 → 15 × 2.0 = 30 点
  史诗技 DamageMul=3.5 → 15 × 3.5 = 52.5 点

目标：enemyHP(12) = 30 × 1.72 ≈ 52 点
校验：单次技能 < enemyHP × 0.8（不一击毙命普通怪）
  → 普通技 18 / 52 = 34%  ✓
  → 稀有技 30 / 52 = 58%  ✓
  → 史诗技 52.5 / 52 = 101% → 接近边界；Cooldown≥25s 限制频率 ✓

单局使用次数估算（以 13min 为基准局长）：
  cd=7s  → 约 111 次（实际受战斗节奏限制，有效约 40-50 次）
  cd=12s → 约 65 次有效机会
  cd=25s → 约 31 次；史诗技高光次数可接受
```

史诗技（DamageMul≥3.5）在 t≥12min 可近似一击击杀单体普通怪，是设计上的高光时刻，通过 `Cooldown≥25s` 控制频率，2 槽无 R 槽保护后依然满足平衡。

## 五、UX / UI 触点

### 5.1 技能槽 HUD 布局

v2.1 HUD 底部居中排列 **2 个**技能槽图标（依赖 [13-UI/HUD](./13-UI与HUD.md) 落地完整布局）：

```
[Q 技能槽]  [E 技能槽]
```

每个图标包含：

| UI 元素 | 说明 |
|---|---|
| 技能图标 | 来自 `SkillConfig.IconPath`，按稀有度边框着色（灰 / 蓝 / 紫）|
| 冷却环 | 顺时针覆盖层，实时显示剩余冷却（纯冷却模式）|
| 充能数字 | 右下角显示 `当前层/最大层`（充能模式），充能层足时数字高亮 |
| 蓄力进度条 | 持续按住时图标底部出现横向进度条（蓄力模式），满格变金色 |
| 键位提示 | 图标上方小字 `Q/E`（可在设置中隐藏）|
| 空槽状态 | 图标区域显示灰色锁形 icon + 提示文字 `SKILL_EMPTY_HINT`（"从宝箱或商人获取技能"）|

**冷却归零反馈**：冷却归零的那一帧，图标播放 0.2s 闪光 + 短促提示音，告知玩家可以使用。

### 5.2 技能命中飘字

技能命中飘字规则沿用 [12-数值平衡 §5.1](./12-数值平衡与曲线.md) 飘字规范；若技能含元素属性，飘字颜色对应元素色。史诗技命中额外播放受击方 0.3s 颜色渲染，强调"大招"感受。

### 5.3 技能获取确认 UI

捡到 / 购买技能时弹出确认面板：

```
+-------------------------------+
|  [技能图标]  火球  ★ 普通     |
|  冷却 7s | 伤害 ×1.2 | 单体    |
|  [预览动效：小火球飞出]        |
|  装备到槽位：[Q][E][放弃]     |
+-------------------------------+
```

玩家主动选择 Q / E 槽位，或选择"放弃"（技能不进背包，直接丢弃）。无背包系统，所见即所得。若目标槽位已有技能，弹出"替换确认"提示，显示当前旧技能信息。

## 六、AI 行为侧需求

### 6.1 Smart Bot（8–10 个）——主动使用技能

Smart Bot 拥有 Q/E 两槽的完整技能决策链：

**决策输入**：
- 当前 2 槽位技能冷却状态（哪个槽 Ready）
- 目标距离与聚集密度（是否值得用 AOE 技能）
- 自身 HP（低 HP 时优先治疗 / 位移技能）
- 当前 Run 时间（早期保守，后期激进）

**决策规则（优先级从高到低）**：

| 优先级 | 条件 | 行为 |
|---|---|---|
| 1 | HP < 30% 且有治疗/位移技能 Ready | 立即释放自保技能 |
| 2 | 3+ 目标聚集在 AOE 半径内 + AOE 技能 Ready | 释放范围技能 |
| 3 | 目标进入攻击距离 + 史诗技 Ready + 冷却 > 20s | 先用史诗技再普攻 |
| 4 | 普通 / 稀有技 Ready + 目标在射程内 | 正常使用 |
| 5 | 全部技能冷却中 | 不用技能，走四件套普攻逻辑 |

**节奏模拟**：Smart Bot 在技能 Ready 后，**延迟 150–350ms（随机）再释放**，模拟人类反应时间。

### 6.2 Light Bot（约 40 个）——不使用技能

Light Bot **不主动使用技能**。若其 build 包含技能槽数据，`BotDecision.UseSkill = false` 始终不触发。左臂纹身仍生效，但 Light Bot 不发 `SkillCastEvent`。

### 6.3 Boss / 精英怪技能 AI

Boss 技能释放由 EnemyModule 按**行为相位**控制：

- 相位 1（HP 100–60%）：每 `BossSkillCooldown_Phase1` 秒释放一次技能，走读 SkillConfig 固定 id
- 相位 2（HP 60–30%）：冷却缩短 30%，技能伤害 ×1.1
- 相位 3（HP < 30%）：冷却再缩短 20%，新增即兴技能（读第 2 个 SkillId 条目）

Boss 技能不受 `ElementOverrideByTattoo` 影响（元素固定）。

## 七、技能获取来源

技能从局内动态获取，初始 0 技能。

| 途径 | 概率 / 频率 | 稀有度分布 |
|---|---|---|
| 普通宝箱 | 每箱 40% 概率含技能 | 70% 普通 / 25% 稀有 / 5% 史诗 |
| 精英宝箱（精英怪掉落）| 每箱 80% 概率含技能 | 40% 普通 / 45% 稀有 / 15% 史诗 |
| **商人购买** | 每局商人备货 1–2 件技能（与颜料 / 武器同列）| 稀有度与当前局时动态挂钩（前 5min 偏普通，后期偏稀有）|
| 事件三选一 | 选项之一为随机技能 | 按当前 Run 时间动态调整 |
| Boss 掉落 | 100% 掉落 1 件技能 | 60% 稀有 / 40% 史诗 |

**不做**：技能树（不符合 BR 局内成长概念）/ 技能合成（超出 MVP 范围）。

**槽位满时获取**：若 2 槽均有技能，可替换任意槽，或选择放弃新技能。商人购买前可预览技能详情，支持与当前槽位技能对比展示。

## 八、风险与开放问题

### 8.1 MVP 技能数量：8 个

8 种技能覆盖 6 种元素 + 3 种充能模型（已在 §四 SkillConfig.json rows 中完整列出）。2 槽位下构筑差异有感知，扩展至 12+ 个技能推迟至 v1.1。

### 8.2 冷却缩减 stat 限制

MVP 阶段**不引入冷却缩减 stat**。左臂纹身只影响 `effectMagnitude`，不修改冷却值。2 槽系统下，若冷却缩减超过 50%，史诗技刷新速度将大幅提升，破坏稀缺感。冷却缩减留作 v1.1 进阶选项，届时需配套 `Cooldown_Min = Cooldown × 0.5` 下限保护。

### 8.3 终极技（Ultimate）不做

MVP 不做终极技，史诗技（高 DamageMul + 长冷却）承担"大招"定位。终极技推迟至 v1.1 评估，届时评估是否绑定 E 槽专属或另起独立机制。
