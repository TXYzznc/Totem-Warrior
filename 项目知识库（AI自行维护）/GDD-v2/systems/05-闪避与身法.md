## 一、玩家体验目标

**一句话**：闪避是"基础生存机制 + 纹身联动放大器"——新手靠它活命，老手用它把左腿纹身的蓄势变成下一击的爆发。

**核心情感**：

- **可信的逃脱感**：按下空格的那一帧，角色身后拖出残影、蓝光一闪、穿过攻击判定，玩家立刻感受到"我闪过去了"——这一帧的视觉信号必须早于"血条有没有掉"先到达玩家眼睛。
- **节奏的锚点**：闪避不是万能键。0.6 秒冷却 + 恢复帧期间的输入缓冲共同形成一个"闪-攻-闪"的节拍，老手会感受到这套节奏，并主动"押着拍打"。
- **左腿纹身的蓄势仪式**：装备了左腿纹身的玩家每次闪避后都会获得一个悬在手上的"下次攻击将附加 XX"的承诺。看着角色身上的纹身微微亮起、等待下一击引爆——这是构筑玩法"从结果到预期"的核心正向反馈。

**反例对照**：
- 闪避后 3 帧才产生 IFrame → 玩家以为自己手慢，永远不信任闪避
- IFrame 无上限 → 全身左腿纹身叠满"无脑闪躲流"，战斗退化为：按空格 → 打一下 → 按空格
- 闪避没有冷却标识 → 玩家不知道还能不能再闪，决策焦虑

**MDA 三层**：
- M（机制）= 空格触发 IFrame + 冲刺 + 冷却 + 左腿纹身 PendingTrigger 拦截
- D（动态）= 玩家在敌人 Active 帧前 200ms 窗口判断是否触发闪避，并决定闪避后是否立即进攻
- A（情感）= **Sensation（闪过攻击的肉体反馈）+ Challenge（择机闪避而非乱按）+ Expression（左腿纹身让每次闪避都"说话"）**

## 二、核心机制

### 2.1 输入触发链

```
玩家按下空格 / 手柄 B 键
    → InputModule.ShouldDodge() = true
    → CombatModule 在当前帧检测到 ShouldDodge
        ├─ 检查冷却（BaseCooldown 剩余 > 0 则忽略，不触发）
        ├─ 触发 DodgePressedEvent（LeftLeg 纹身订阅，打包 PendingTrigger）
        ├─ 玩家角色进入 IFrame（DodgeIFrames 帧）
        ├─ 执行短距离冲刺（DashDistance × WASD 输入方向，无输入则向角色面朝方向）
        └─ 重置冷却计时器（BaseCooldown 秒）
```

**输入缓冲**：5 帧（83ms）——闪避绝不允许"我按了但没动"（来自 02-战斗手感 §2.1 规范）。

### 2.2 闪避三段帧

```
Frame  1–N    IFrame         [完全无敌，hitbox 对玩家关闭]
Frame N+1–N+4 Recovery       [无敌结束，仍在冲刺弧线末端，可输入缓冲普攻]
冷却计时器     BaseCooldown   [从 Frame 1 开始计时，倒计完毕前不可再次闪避]
```

| 左腿纹身等级 | DodgeIFrames（帧） | IFrame 时长 | 冲刺距离 |
|---|---|---|---|
| 未刻（0 槽） | 8 | 133ms | 3.0 m |
| 1 槽（白色基础） | 10 | 167ms | 3.2 m |
| 2 槽（同色共鸣初步） | 12 | 200ms | 3.4 m |
| 3–4 槽（中期） | 14 | 233ms | 3.6 m |
| 6 槽全同色（化身） | 16 | 267ms | 4.0 m |

**IFrame 硬上限**：18 帧（300ms）—— 超过此值视为"无脑闪躲流"退化，由 `BalanceConstantsConfig.DodgeIFrameMax` 锁死（[12-数值平衡 §四](./12-数值平衡与曲线.md)）。

**冲刺方向优先级**：
1. 玩家有 WASD 方向输入 → 冲刺至输入方向
2. 无方向输入 → 沿角色当前面朝方向冲刺
3. 禁止向后冲刺时判断为"后向闪避"——当前 MVP 不区分方向，统一处理

### 2.3 动作取消窗口

闪避是四件套中**最高优先级**，可在以下阶段取消：

| 当前阶段 | 是否可被闪避取消 |
|---|---|
| 普攻 Startup（1–4 帧） | 可取消 |
| 蓄力 Charging（1–24 帧） | 可取消（蓄力中断，进度归零） |
| 普攻 Recovery（8–15 帧） | 可取消 |
| **普攻 Active（5–7 帧）** | **不可取消**（hitbox 已出，取消会造成伤害判定丢失）|
| 技能释放期间 | 取决于技能 `CancelableByDodge` 属性（由 04-主动技能定义）|

### 2.4 左腿纹身联动（InterceptApply / PendingTrigger）

这是闪避系统与纹身系统耦合的核心节点（[01-纹身构筑 §2.2](./01-纹身构筑系统.md)）：

```
DodgePressedEvent 触发
    → LeftLeg 纹身策略订阅
    → 本次不立即释放元素效果
    → 打包为 PendingTrigger，挂载在玩家身上
    → 玩家下次普攻命中时（OnAttack），PendingTrigger 随普攻一同释放
        └─ 效果 = 左腿(颜色)(图案) 的完整元素效果 × scaleFactor(左腿)
```

**PendingTrigger 规则**：
- 每次新闪避覆盖旧 PendingTrigger（不叠加，只保留最新一次）
- PendingTrigger 有效期 = 5 秒（超时自动失效，避免"囤积"策略）
- PendingTrigger 激活后视觉提示：角色手部出现左腿纹身颜色的微弱光晕，表示"下击附加"

### 2.5 冷却与双闪禁止

| 参数 | 值 | 说明 |
|---|---|---|
| BaseCooldown | 0.6 s | 未刻左腿纹身时的基础冷却（帧层面 = 36 帧） |
| 左腿被动冷却缩减 | 见 DodgeConfig.json §四 `CooldownReductionPerSlot` | 每加一槽最多缩减 0.05s，6 槽满上限缩减 0.3s → 最短 0.3s |
| 双闪禁止 | 单次冷却内禁止再次触发 | `CombatModule` 维护 `dodgeCooldownTimer`，> 0 时 ShouldDodge 强制返回 false |

**设计意图**：最短 0.3s 冷却（18 帧）= 5 闪/秒理论上限，但实际受 Recovery 帧（4 帧）+ 人类操作节奏限制，真实上限约 3 闪/秒，不会退化为"空格连点闪躲流"。

> 修改此文件后运行 Unity 菜单 `Tools/DataTable/生成全部配置表代码`，生成 `Assets/Scripts/DataTable/DodgeConfig.cs`。

**IFrame 曲线公式**（落地实现参考）：

```
DodgeIFrames(slotLevel) = clamp(
    BaseIFrames + IFramePerSlot × slotLevel,
    BaseIFrames,
    DodgeIFrameMax
)

DodgeCooldown(slotLevel) = max(
    BaseCooldown - CooldownReductionPerSlot × slotLevel,
    MinCooldown
)

DashDistanceFinal(slotLevel) = DashDistance + DashDistancePerSlot × slotLevel
```

| 左腿槽位数 | IFrame（帧） | IFrame（ms） | 冷却（s） | 冲刺距离（m） |
|---|---|---|---|---|
| 0 | 8 | 133 | 0.60 | 3.0 |
| 1 | 10 | 167 | 0.55 | 3.2 |
| 2 | 12 | 200 | 0.50 | 3.4 |
| 3 | 14 | 233 | 0.45 | 3.6 |
| 4 | 16 | 267 | 0.40 | 3.8 |
| 5 | 18（上限） | 300 | 0.35 | 4.0 |
| 6 | 18（上限） | 300 | 0.30（下限）| 4.2 |

**平衡检验点（balance-check 输出）**：
- 0 槽时 IFrame 133ms = 玩家对 200–400ms 攻击的容错窗口 → 合理（新手需提前 67ms 预判）
- 6 槽满 IFrame 300ms + 最短冷却 300ms → 理论最高闪避密度 = 1 次 / (300ms + 200ms 冲刺) ≈ 2 次/s，DPS 无显著损失
- PendingTrigger 5s 有效期 → 正常对局 50% 概率在 2 次普攻内消耗，不构成"囤积"退化路径

## 五、UX / UI 触点

### 5.1 闪避残影

IFrame 期间角色生成 3 个半透明残影沿冲刺方向拖尾：

- 残影颜色 = 当前左腿纹身主色（未刻 = 半透明白色；金腿 = 金色；蓝腿 = 冰蓝色）
- 残影透明度从 60% 渐变至 0%，最后一帧消失
- 同 06-角色设定 §六"远观可读性"——PVP 中看到对手残影颜色即可判断其左腿纹身元素
- 残影由 client-ta 负责（URP 后处理，不增加 draw call）

### 5.2 IFrame 期间的蓝光提示

闪避开始的第 1 帧：角色轮廓出现微弱蓝色边缘光（Edge Glow），持续 IFrame 全程：

- 蓝光叠加左腿纹身颜色（纯蓝 = 无纹身；混合色 = 有纹身）
- 视觉目的：玩家自己能清楚感知"我现在是无敌的"，不依赖记忆帧数

### 5.3 PendingTrigger 激活提示

闪避命中 `DodgePressedEvent` 且 PendingTrigger 已打包时：

- 角色持武器的手部出现左腿纹身颜色的微弱光晕（持续至 PendingTrigger 释放或超时）
- HUD 上闪避冷却条旁出现一个小 icon（颜色 = 左腿纹身元素色）表示"蓄势已就绪"
- 玩家下次普攻命中时，PendingTrigger 同步释放，光晕熄灭 + 元素爆发 VFX 出现

### 5.4 闪避冷却标识（HUD）

位置：右下角，与闪避相关的短横条（02-战斗手感 §五 HUD 元素中已定义）：

- **就绪状态**：横条满色（左腿纹身颜色，无纹身则白色）
- **冷却中**：横条从左向右消耗，冷却完毕闪烁一次提示"可用"
- **无冷却时不显示**（避免 HUD 常驻过多元素，详见 13-UI/HUD GDD）

## 六、AI 行为侧需求

### 6.1 Smart Bot（8–10 个）

Smart AI 走完整闪避逻辑，包含 PendingTrigger 联动：

| 决策条件 | 行为 |
|---|---|
| 检测到对方角色进入 Active 帧（hitbox 激活） | 触发 `ShouldDodge = true`，方向 = 远离攻击来源 |
| 检测到对方蓄力（Charging 帧 ≥ 18 帧时判定为"蓄力中"） | 预判 Active 帧时机，提前 8–12 帧触发闪避（模拟人类预判，非精确帧） |
| 远程投射物进入 4m 内 | 向垂直方向闪避（侧闪规避） |
| PendingTrigger 已就绪 | 优先寻找攻击机会（`ShouldAttack` 优先级提升），以便释放蓄势 |

**抖动模拟**：Smart AI 的闪避触发时机加入 ±2 帧（±33ms）随机偏差，防止被识别为机器人手感（16-BotControllerModule 详设落地）。

### 6.2 Light Bot（40 个）

Light AI 只在特定条件下被动触发闪避，不做帧判断：

| 条件 | 触发 | 备注 |
|---|---|---|
| 当前 HP < 30%（低血量） | 50% 概率触发闪避 | 每 1.2s 检测一次，不连续触发 |
| 受到单次伤害 > 当前 HP × 20% | 30% 概率触发闪避 | 反应型被动，非预判 |
| 其他情况 | 不触发闪避 | Light AI 以普攻为主 |

Light AI **不处理 PendingTrigger**——即使其左腿有预设纹身，PendingTrigger 的拦截逻辑仍正常运行（TattooModule 处理），但 AI 不会主动"等待释放"。

## 七、风险与开放问题

### 7.1 闪避方向（WASD 输入方向 vs 鼠标方向）

**问题**：玩家不输入方向时闪避方向是否跟随鼠标朝向（面朝方向）还是 WASD 移动方向？

| 方案 | 优 | 劣 | 推荐 |
|---|---|---|---|
| A. 优先 WASD，无输入则面朝 | 与移动手感一致，直觉性强 | 站桩远程时无法"回闪" | — |
| B. 始终跟随鼠标面朝方向 | 鼠标玩家掌控感更强 | 移动中鼠标朝向与移动方向分离，反直觉 | — |
| C. WASD 优先，鼠标附加微调（±45°）| 两种模式兼顾 | 实现较复杂 | — |
| **D. WASD 优先，无输入则背向目标**（近战直觉）| **近战场景最直觉**（退步闪） | 远程战无目标时退化为随机 | **MVP 推荐** |

**当前推荐**：MVP 走方案 D——有 WASD 输入时按输入方向，无 WASD 且有锁定目标时背向目标闪避。无 WASD 且无目标时回落为面朝方向。需 playtest 验证后可调整。

### 7.2 落地缓冲（闪避补帧）

**问题**：冲刺结束时如果角色停在斜坡或地形边缘，是否需要额外的"落地缓冲帧"来修正位置？

**当前**：MVP 不做落地缓冲，冲刺为地面平移（不含垂直位移），碰撞由物理层自动处理。

**若出现卡墙 / 位移过冲问题**：增加 `DashEndPushBack = 0.1m` 安全退出距离，由 client-unity 在 `CombatModule` 冲刺位移计算末段处理，不影响 IFrame 时序。

### 7.3 双闪避取消策略

**推荐**：单次冷却内禁止再次触发，无"双闪"机制。

**理由**：
1. 双闪会使 IFrame 覆盖率接近 50%（300ms × 2 / (600ms + 300ms)），接近"不死"
2. 与 PendingTrigger 机制冲突（双闪会覆盖上次 PendingTrigger，消耗纹身蓄势而不释放）
3. 节奏感：单闪 + 进攻 + 单闪的"闪-打-闪"节奏比"双闪躲开 → 随意打"更有策略深度

**若后期扩展**：可在特定左腿纹身图案（如"流线"）上追加"二段闪"被动，而不是全局开放，以此区分构筑差异。

### 7.4 IFrame 与 Hitstop 的交互

**问题**：若玩家在 IFrame 期间目击到敌人暴击（触发 Hitstop），IFrame 的绝对帧数是否受 Hitstop 时间缩放影响？

**规则**：Hitstop 的时间缩放作用于画面 + VFX + 双方角色，但**不压缩 IFrame 计时**——IFrame 帧数以逻辑帧（FixedUpdate 60fps）为单位计，Hitstop 只是画面层的慢镜头。这样"Hitstop 中玩家的 IFrame 不会被意外吞掉"。

见 [02-战斗手感 §7.4 50 actor 同场 Hitstop 性能](./02-战斗手感.md) 的处理前提。
