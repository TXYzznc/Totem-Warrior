## 一、玩家体验目标（30 秒读懂）

**一句话**：单局 10-15 分钟、60fps 不掉帧、攻击有重量、闪避有节奏；按下按键的那一帧，手要"信"屏幕在响应你。

**核心情感**：
- **重量感** —— 普攻不是"鼠标点一下角色挥一下"，而是"挥出去那一帧画面会顿一下、敌人会被推出一截、屏幕角落抖了 2 像素"。
- **节奏感** —— **四件套（普攻 / 蓄力普攻 / Q+E 双技能槽 / 闪避）**的输入帧窗口是被精心设计过的"乐谱"。新手随便按也不会卡死手感；老手能押着帧打出"闪避取消蓄力"或"Q 起手 → 闪避取消 → E 收尾"的节奏型。
- **决策可读** —— 屏幕上任何一次伤害结算都能在 100ms 内被反馈给玩家——飘字、震动、Hitstop、目标受击姿态。

**单局节奏收紧（v2.1 核心改动）**：
- 单局 10-15min（v2 隐含 ~20min）→ 玩家每秒钟都在做小决策
- 普攻基础周期由 15 帧 → **13 帧**（约 217ms，4.6 击/秒）
- 两个技能槽（Q/E）冷却由 v2 单技能 12-18s 区间 → **8-12s 区间**，配合双槽 = 平均 5-6s 就有 1 次技能可放
- 闪避冷却保持 200ms 上限不变（手感优先）

**反例对照**：
- 如果 3 技能槽塞 1/2/3 数字键，玩家眼睛要在 HUD 和场景之间反复扫 → **三键认知负担在 10min 紧凑节奏里会爆**。Q/E 是 FPS / MOBA 通用肌肉记忆，左手不离 WASD。
- 如果技能 CD 还按 v2 的 12-18s 区间设，单局 12min 玩家平均只能用 4-6 次技能 → build 强度感受不到。
- 如果普攻还是 15 帧，10min 内的输入密度不够 → 玩家觉得"打了好久还没结束"。

**MDA 三层**：
- M（机制）= 四件套 × 三段帧 × Hitstop × IFrame × **2 技能槽 CD 错峰**
- D（动态）= 玩家在 0.3s 窗口内做 "普攻 / 蓄力延后 / Q 起手 / E 收尾 / 闪避取消" 五选一决策
- A（情感）= **Sensation（感官）+ Challenge（精进）+ Submission（顺从节奏）+ Fellowship（双技能编排乐趣）**

## 二、核心机制 —— 战斗四件套（v2.1 重定义）

### 2.1 输入帧规范

所有输入走 `InputModule` → `IPlayerController.ShouldXxx()` 轮询（[CONTRACT §三](../../../openspec/changes/05-gdd-v2-full-design-docs/CONTRACT.md)）。

| 输入意图 | 触发条件 | 输入缓冲（帧）| 备注 |
|---|---|---|---|
| `ShouldAttack` | 鼠标左键按下 / 手柄 X | 3 帧 | 恢复帧内按下也算下一段连击 |
| `ShouldChargedAttack` | 鼠标左键长按 ≥ `ChargedReadyThreshold` 后松开 | 5 帧 | 满蓄一帧 = 边缘金色脉冲 |
| `ShouldUseSkill(slot)` | **Q 键（slot=0）/ E 键（slot=1）** | 2 帧 | **v2.1：数字键 1/2/3 → Q/E 双槽** |
| `ShouldDodge` | 空格键 / B 键 | 5 帧 | 极宽缓冲 |
| `ShouldInteract` | F 键（v2.1：E 让位给技能槽） | 0 帧 | 拾取严格按下瞬间 |

**v2.1 键位变更**：
- 主动技能：Q（左槽） / E（右槽）—— FPS/MOBA 通用，左手不离 WASD
- 拾取/交互：**F 键**（v2 的 E 键让位）
- 数字键 1/2 仍可作备用绑定（设置里可改），但 HUD 默认显示 Q/E

### 2.2 三段式攻击帧（基础公式）

公式不变（参 v2）：

```
StartupFrames  = BaseStartup  × (1 - AttackSpeedBonus)
ActiveFrames   = BaseActive   × (1 - AttackSpeedBonus × 0.5)
RecoveryFrames = BaseRecovery × (1 - AttackSpeedBonus)
```

**普攻基线（v2.1 短刀，节奏收紧）**：

| 段 | v2 帧数 | **v2.1 帧数** | 时长 | 设计意图 |
|---|---|---|---|---|
| Startup | 4 | **3** | 50ms | 取消窗口仍在，反馈更快 |
| Active | 3 | **3** | 50ms | hitbox 不缩 |
| Recovery | 8 | **7** | 117ms | 略缩，连击更紧 |
| **合计** | 15 | **13** | **217ms** | **4.6 击/秒**（v2 是 4 击/秒） |

**蓄力普攻（v2.1 阈值压缩）**：

- 长按 `ChargedReadyThreshold = 20 帧`（333ms，v2 是 24 帧）后进入"蓄满"
- 蓄满后再持续按 24 帧（400ms，v2 是 30 帧）进入"过载"
- 蓄力收紧的理由：单局 10-15min 内，玩家需要更频繁地把蓄力打出来（之前 v2 实测玩家在紧节奏里普遍跳过蓄力）

### 2.3 Hitstop（不变）

参 v2，4 档帧数 `[2, 4, 5, 7]` 不变——反馈层不动，节奏只压缩动作层。这是 §七 5.1 的设计哲学延续。

### 2.4 闪避无敌帧（IFrame，不变）

参 v2 §2.4 表与公式。上限 18 帧。

### 2.5 帧表（v2.1 短刀 + 未刻纹身 baseline）

```
普攻：
  Frame  1-3   Startup     [可被闪避取消]
  Frame  4-6   Active      [hitbox on, Hitstop 触发]
  Frame  7-13  Recovery    [可输入缓冲下一击 / 技能 / 闪避]
  总计 13 帧 = 217ms

蓄力普攻：
  Frame  1-20  Charging    [可松开取消, 可被闪避/技能取消]
  Frame 21     Ready       [满蓄一帧提示]
  Frame 21-44  Overcharge  [过载窗口]
  松开：
  Frame  1-3   Startup
  Frame  4-7   Active      [×1.5 伤害]
  Frame  8-19  Recovery

闪避：（不变）
  Frame  1-8   IFrame
  Frame  9-12  Recovery
  总计 12 帧 = 200ms = 5 闪/秒上限

主动技能（Q/E 通用骨架，各技能详表 → 04-主动技能）：
  Frame  1-N   Startup     [N 视技能, 8-15 帧]
  Frame N+1   Active      [hitbox / 投射物生成]
  Frame ...    Recovery    [12-25 帧]
  CD 8-12s    [双槽错峰：平均 5-6s 玩家就有一个技能可放]
```

## 五、UX / UI 触点（影响 13-UI v2.1）

### 5.1–5.5 屏幕震动 / 飘字 / 暴击高亮 / 闪避残影 / 蓄力反馈

参 v2，**仅蓄力时序调整**：

- Frame 1-10（蓄力 0-50%）：武器边缘淡色光晕
- Frame 11-19（蓄力 50-99%）：光晕脉冲，音效升调
- Frame 20（蓄满那一帧）：**金色脉冲 + "叮" 音效**
- Frame 21-44（过载窗口）：光晕变红 + 轻微震屏

### 5.6 HUD 元素（v2.1 重设）

| 元素 | 位置 | v2 内容 | **v2.1 内容** |
|---|---|---|---|
| HP 条 | 左下 | 头像 + 数字 + 血条 | 不变 |
| 蓄力指示 | 角色头顶 0.5m | 圆形 progress | 不变 |
| 闪避冷却 | 右下 | 短横条 | 不变 |
| **技能槽** | **屏幕底部居中** | **3 槽，1/2/3 提示** | **2 槽（Q/E），间距加大** |

**v2.1 HUD 设计意图**：
- 2 槽视觉中心化（屏幕正下方居中，两槽对称排布），CD 进度条更显眼
- Q/E 键提示字符更直观（vs 数字 1/2/3 玩家眼睛要数）
- 节省的 HUD 空间留给 **build 增益堆叠图标**（左臂纹身的"技能触发计数"飘字提示需要在 HUD 上有视觉锚点）

→ **传递给 13-UI 与 HUD GDD v2.1**：技能槽数 3→2、键位提示由数字 → Q/E、布局重排。

## 六、AI 行为侧需求（2 槽更易调优）

**Smart AI（8-10 个）**：完整四件套——

| 行为 | 决策依据 |
|---|---|
| 普攻 | 目标在 hitbox 范围内 |
| 蓄力 | 目标 HP 高 / 自身 build 含右臂蓄力强化 |
| **技能（Q/E）** | **v2.1：2 槽决策模型 = 简单二维判断（Q 起手 vs E 收尾），比 v2 的 3 选 1 状态机降低 33% 复杂度，Smart AI 更易调出"像人"的节奏** |
| 闪避 | 检测到对方 Active 帧 / 投射物入场 |

**Light AI（40 个）**：仅普攻 + 闪避，不变。

**关键验收点**：Smart AI 的技能 CD 触发分布要呈现"双峰"（Q 用得早、E 用得迟），不是"打开就放双技能" → 这是 16-BotControllerModule 详设里 `BotBuildPlanner` 的输出约束。

## 七、风险与开放问题

### 7.1 ⭐ 单局 10-15min 是否真的能撑住"build 探索"乐趣？

**问题**：v2 单局 ~20min 是给玩家"刻 3-4 次纹身、试 1-2 套 build"的时间预算。压到 10-15min 后玩家可能只刻 2 次纹身就被推进 Boss → build 体验缺失。

**缓解**：
- 纹身刻印动作从 v2 的"开纹身师 UI 操作 ~30s" → 简化为"局内拾取纹身贴纸即时套用"（详 09-纹身师 NPC v2.1）
- 单局推荐 3 纹身刻印事件（开局 / 中段 / Boss 前），节奏由 08-宝箱与探财节奏 v2.1 落地

**风险等级**：中。需要 playtest 验证 12min 玩家能否完成 1 套 build 闭环。

### 7.2 移动端 Q/E 键怎么映射？

**问题**：移动端无键盘，2 技能槽需占 HUD 屏幕空间。

**方案**：
- 移动端右下放 2 个圆形技能按钮（替代 v2 的 3 槽圆盘）
- 配合右下蓄力按钮（参 v2 §7.1 方案 C）= 总共 3 个动作按钮 + 1 闪避
- 屏幕右下区域比 v2 的 4 按钮（3 技能 + 1 蓄力）布局更宽松

**接受**：移动端按钮总数 4 → 4，但布局更宽松。

### 7.3 蓄力阈值压到 20 帧是否会误触？

**问题**：v2 的 24 帧（400ms）是"用户长按 vs 点击"的安全阈值。压到 20 帧（333ms）可能被部分玩家的"双击连点"误触发蓄力。

**缓解**：保留 v2 的输入缓冲 3 帧——按下到松开 < 17 帧（283ms）算单击普攻而非短蓄力。playtest 后视情况微调到 18-22 帧之间。

### 7.4–7.5 50 actor 性能 / 攻速极限破坏节奏

参 v2，不变。Hitstop 不受攻速影响是设计哲学的核心，v2.1 节奏收紧不破坏该原则。
