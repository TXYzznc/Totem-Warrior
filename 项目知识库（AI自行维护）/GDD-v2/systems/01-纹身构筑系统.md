## v2.1 修订速览

v2.1 把"build 构筑"从 v2 的纯抽象组合**前移到玩家身体动作**：每一个纹身都必须由玩家本人**蹲下来花 3-8 秒亲手刻**，期间无法移动、无法攻击、被打就作废。纹身师不再是刻图的人——他变成"花稀有颜料赌随机词缀"的附魔工（见 09 GDD）。

变更要点：

1. **新增 §2.8「自纹身读条流程」**：核心机制从"打开界面选三元组 → 落实"变为"打开界面 → 读条 3-8 秒 → 落实"。读条 = 风险时刻，是单局节奏中的关键决策点。

5. **§五 UX** 新增"自纹身界面线框"和"读条 UI"（吃鸡风格 3-8s 条 + 中断震屏）。
6. **§七 风险**新增 v2 → v2.1 迁移成本评估。

## 一、玩家体验目标（30 秒读懂）

**纹身就是这个游戏的"角色养成"——但 v2.1 之后，纹身**首先是一场身体劳作**，然后才是 build。**

v2 的承诺是"build 翻译成身体语言"。v2.1 把这件事推得更狠：**你不仅要看到自己的 build 写在身上，你还要亲手在战场上刻它**。任何地点按 Tab，弹出工作台，选好部位/颜色/图案，开始读条——脑袋 8 秒，躯干 8 秒，手臂 5 秒，腿 3 秒。期间你不能动、不能打、被攻击就作废、扣 50 金币。

这一层读条不是"摩擦税"，而是把 build 决策塞进**与战斗同一个时间轴**里的核心张力：

1. **"刻在哪里"是空间决策**：野外刻效率高但风险高；缩圈前回到安全屋刻安全但浪费时间；和队友配合时让队友 cover 你刻——纹身因此成为**关系建立**的一部分。
2. **"刻什么部位"是节奏决策**：腿部 3 秒、快、低收益（移动 + 闪避增益）；脑袋 8 秒、慢、高收益（暴击爆发）——单局 10-15 分钟的玩家基线刻满 4-5 部位，意味着每一次"花 8 秒刻脑袋"都是一次"放弃 2 次刻腿"的机会成本。
3. **"敌人在刻"是猎物信号**：远远看到一个角色不动、身上有读条圈——你立刻知道有 3-8 秒可以扑过去打断。读条变成战场上的**可观察战术信号**。
4. **死亡即遗产**：你在身上的纹身死亡时不脱落，但所有**未使用的颜料 + 配方碎片**会以"拓本"形式留在死亡宝箱里，杀你的人可以继承你正在筹备的那套 build。

**反例（v2 没有读条的世界）**：玩家在背包界面里"勾选部位/颜色/图案 → 确认"，build 瞬间生效。这版本的失败在于：（a）刻纹身和战斗是两个时间维度，build 决策没有进入战斗节奏；（b）所有玩家都会在安全屋慢慢配 build，不存在"野外赌一把"的紧张；（c）猎人/猎物的对称信号消失了——你不知道对手在做什么。读条把这三件事一次性解决。

> **MDA 三层**：
> - **Mechanics** = 6×7×8 笛卡尔积 **+ 自纹身读条 3-8s 中断惩罚 + 配方解锁 4 渠道 + 颜料三档**；
> - **Dynamics** = 玩家在 10-15 分钟一局内**在战斗节奏中插入 4-5 次读条窗口**，逐步显形一套 build 并据此调整打法；
> - **Aesthetics** = **Expression（表达）+ Discovery（发掘）+ Fellowship（被认出来）+ Tension（读条 = 脆弱时刻）**。

## 二、核心机制

### 2.1 三层正交结构（v2 不变）

任何一个纹身槽都由 `(部位, 颜色, 图案)` 三元组决定。规则不变：

> **当 [部位.触发事件] 发生时，以 [图案.形状] 的方式释放 [颜色.元素] 效果，强度按 [部位.缩放属性] 计算。**

### 2.2 6 部位 / 2.3 7 颜色 / 2.4 8 图案

完整表格沿用 v2，此处不重抄（参考 [v2.0 历史快照](#)，机制零变更）。核心要点：

- **6 部位**：脑袋（OnCrit, ScaleFactor=10.0）/ 躯干（OnDamaged, ScaleFactor=0.12, 反弹）/ 左臂（OnSkillCast, 0.6）/ 右臂（OnAttack, 0.8）/ 左腿（OnDodge, 40.0, 拦截打包）/ 右腿（OnMoveTick, 1.6, 前向重塑）
- **7 颜色**：红/黄/绿/蓝/紫/金/白；白色 ColorMultiplier = 1.20，其余 1.0
- **8 图案**：直线 / 圆环 / 螺旋 / 锯齿 / 闪电 / 星形 / 流线 / 兽形

### 2.5 联动机制 / 2.6 被动属性

沿用 v2 §2.6 / §2.7 不变：同色共鸣（3+/4-5/6 同色三档）/ 同图案呼应 / 对称构型 / 颜色反应（红+黄爆裂、蓝+绿寒毒、紫+金混乱、白+任放大）；被动 stats = 部位.PassiveDimension × 颜色.Element × 图案.PassiveStrength。

### 2.7 两个具体例子

**例 1：右臂 × 黄 × 闪电（"链电拳"）** — 每次普攻命中，电流跳 3 跳（衰减 0.7×），每跳麻痹 0.5s。
**例 2：躯干 × 红 × 圆环（"火爆反弹甲"）** — 受伤瞬间胸口炸开火浪扑向攻击者 + 周围 4 个 ×60%，全员着火。

### 2.8 ⭐ 自纹身读条流程（v2.1 新增）

**v2.1 之前**：玩家走到纹身师 NPC → 弹出界面 → 选 → 即时落实。
**v2.1 之后**：玩家**任意地点**按 `Tab` → 弹出界面 → 选 → **读条 3-8s** → 落实。纹身师不再参与刻字（详见 09 GDD）。

**触发条件（同时满足）**：

```
1. player.inkInventory[colorId]   ≥ INK_CONSUME_COUNT(colorId)   // 颜料够
2. player.recipeUnlocked(colorId, patternId) == true             // 图案配方已解锁
3. player.tattoo.partHasEmptySlot(partId) == true                // 该部位有空槽
```

任一不满足 → 工作台对应选项灰显并显示原因（"颜料不足"/"配方未解锁"/"部位已满"）。

**读条时长（按部位）**：

| 部位 | 读条时长 (s) | 设计意图 |
|---|---|---|
| 脑袋 | **8** | 高收益（暴击爆发 SF=10）配对高读条风险 |
| 躯干 | **8** | 反弹机制需要"投资姿态"——你要长时间脆弱才能换来未来的反弹甲 |
| 左臂 | **5** | 中等收益（技能强度 SF=0.6） |
| 右臂 | **5** | 中等收益（普攻持续 SF=0.8） |
| 左腿 | **3** | 低收益但高使用频率，快读条让玩家敢于野外刻 |
| 右腿 | **3** | 同左腿 |

**读条期间状态约束**：

| 行为 | v2.1 处理 |
|---|---|
| 移动输入 | **忽略**（角色冻结） |
| 攻击 / 技能输入 | **忽略**（按键被读条 UI 拦截） |
| 受到任何伤害 | **立即中断** |
| 被击杀 | **立即中断 + 死亡** |
| 主动取消（再按 Tab） | **立即中断**（视为玩家自愿放弃） |

**完成 / 中断结果**：

| 结果 | 颜料 | 金币 | 配方 | 事件 |
|---|---|---|---|---|
| 完成 | 扣 `INK_CONSUME_COUNT` 瓶 | 不扣 | 不消耗（配方是永久的） | `TattooFinishedEvent { Owner, NewSlot }` → `TattooModule.Equip` |
| 中断（任何原因） | **不扣** | **扣 50** | 不消耗 | `TattooCancelledEvent { Owner, Reason }` |

**核心设计原则**：

1. **颜料不扣**是关键——v2.1 不能让玩家"试一次失败损失 1 瓶稀有颜料"，那会让自纹身变成劝退机制。**金币扣 50 是有意为之的轻摩擦**，让中断有"啊可惜"的体感，但不会破坏 build 计划。
2. **TattooModule.Equip 接口零改动**：已交付的实现层不需要重构，v2.1 只在 UX 层加一道读条前置。读条完成后调用同一个 `Equip(partId, colorId, patternId)`。
3. **读条 = 战场信号**：远处玩家看到你身上有读条圈（圆形进度条 + 蓝色发光），就知道你 3-8 秒内是肉鸽。这是核心的猎物/猎人对称机制。

### 2.9 ⭐ 初始配方与解锁渠道（v2.1 新增）

**起手 2 配方**：新角色出生时持有 `直线` + `圆环` 两个图案配方（覆盖所有 7 种颜色组合，但**颜色仍受颜料库存约束**）。

> 设计意图：直线 = 单点 1 次（最直觉），圆环 = 主目标 + 4 个 × 60%（学习"图案改变命中模式"的最佳教学样本）。新人第一局只用这两个配方就能玩出 6×7×2 = **84 个独立组合**，足够学习三层正交结构而不被 336 全空间淹没。

**其余 6 个图案配方解锁的 4 条渠道**：

| 渠道 | 范围 | 节奏感 | 设计意图 |
|---|---|---|---|
| **① 局外解锁** | 永久解锁，跨局保留 | 慢（成就 / 累计 Run 数 / 击杀里程碑触发） | 给长期玩家的"图鉴感"成长，新号到老号配方越来越多 |
| **② Boss 掉落** | 永久解锁（首杀） / 重复掉落时转化为 1 张完整配方道具 | 中（每局 1-2 个 Boss） | 把 Boss 战的奖励曲线和 build 深度绑死 |
| **③ 宝箱 / 事件** | 局内有效（配方碎片 3→1 完整配方）或永久（精英箱 / Boss 箱直接掉完整配方） | 中频（贯穿单局 10-15 min，详见 08 GDD §2.2-2.3） | 提供"局内抓配方 → 当场用上"的快爽点 |
| **④ 死亡玩家拓本** | **仅本局有效**（不进局外永久库） | 高方差（杀人 / 收尸触发） | 让"再玩一局" + "PVP 抢资源"是同一件事 |

**颜料三档**：

| 档位 | 颜色 | 来源密度 | 典型用法 |
|---|---|---|---|
| **常见** | 红 / 黄 / 绿 / 蓝 | 普通宝箱主产出（50-70% 概率） | 早期保底 build，1-3 槽位用它 |
| **稀有** | 紫 / 金 | 精英箱 / Boss 箱 / 商人售卖（10-25% 概率） | 中后期 4-5 槽位升级，或留给 09 附魔花费 |
| **传说** | 白 | Boss 箱 / 死亡宝箱（5% 以下） | 终局 6 槽位 + 白色 1.20× 的核心爆发 |

颜料瓶定义与掉落表完整落在 [08-宝箱与探财节奏 §2.1-2.3](./08-宝箱与探财节奏.md)；本 GDD 仅引用档位语义。

**单局基线**：玩家在 10-15 min 一局内、**不死不杀**的情况下，应能刻满 **4-5 个部位**（消耗 4-5 瓶颜料，对应 08 GDD 中早+中期累计 5-8 瓶颜料的产出曲线）。激进玩法（杀人抢拓本 / 进 Boss 房）可推到 6 槽满。

## 五、UX / UI 触点

### 5.1 自纹身界面（v2.1 新增，任意地点 Tab 唤出）

```
+----------------------------------------------------------+
|  纹身工作台                              [×] (再按 Tab 关) |
|  ----------------------------------------------------------|
|  [角色立绘 + 6 部位高亮选择]                                 |
|                                                            |
|  选中部位：右臂  ▼ (有空槽 ✓)   预估读条：5.0s              |
|                                                            |
|  ┌─ 颜色（库存可用才亮）────────────────────────────────┐  |
|  │  ● 红 (×3)  ● 黄 (×1)  ○ 绿 (×0灰)  ● 蓝 (×2)        │  |
|  │  ● 紫 (×1)  ○ 金 (×0灰)  ○ 白 (×0灰)                 │  |
|  └────────────────────────────────────────────────────────┘  |
|                                                            |
|  ┌─ 图案（配方未解锁的禁用）────────────────────────────┐  |

|  │  ⭐星形🔒  〰️流线🔒  🐉兽形🔒                          │  |
|  └────────────────────────────────────────────────────────┘  |
|                                                            |
|  ◆ 预览：右臂 × 黄 × 直线 = "黄电单点拳"                     |
|  ◆ 消耗：黄颜料 ×1                                          |
|  ◆ 风险：读条 5.0s 期间被打 → 颜料返还、扣 50 金币           |
|                                                            |
|  [ 开始绘制（5.0s 读条）]                  [取消]            |

```

**关键 UX 原则**：

1. **库存即可见**：颜色按钮直接显示库存数量（×N）；为 0 自动灰显。
2. **配方锁可见**：未解锁图案不隐藏而是显示锁图标，告知"这个图案存在，但还没拿到配方"，刺激探索动机。
3. **预览必现**：选完未确认前用一行文字 + 小动效模拟（"普攻附带火焰链跳 3 段"），不是数字。
4. **风险显式告知**：每次确认前明确写出读条时长 + 中断惩罚 50 金币。

### 5.2 读条 UI（v2.1 新增，角色头顶 + 屏幕中央）

吃鸡风格：

```
        ┌─────────────────────────┐
        │  正在刻：右臂 × 黄 × 直线 │
        │  ▓▓▓▓▓▓▓░░░░ 3.2 / 5.0s  │
        │  [按 Tab 取消]           │
        └─────────────────────────┘
                  ▲ 角色脚下圆形进度条同步显示
                    （蓝色光圈 → 远处敌人可见）
```

**关键反馈**：

- 中断时屏幕震屏 + 红色闪烁 + 音效"叮"破碎声 + 飘字"-50 金币 / 颜料返还"
- 完成时角色身上对应部位发光 0.5s + 飘字"右臂 已刻：黄 × 直线"

## 六、AI 行为侧需求

### 6.1 AI 也必须走读条（v2.1 新增约束）

智能 AI（8-10 个）调用 `TattooModule.Equip` 时**同样触发 3-8 秒读条**，发布 `TattooFinishedEvent`。设计意图：

- 保持"读条 = 战场可观察信号"的对称性——玩家看到 AI 在读条同样可以扑过去打断
- 避免 AI 瞬间满 6 槽位形成对玩家的不公平体验
- BotBuildPlanner 决策频率 = 30s 重思一次，配合 3-8s 读条意味着 AI 单局最多刻 5-7 次（与玩家基线接近）

### 6.2 BotBuildPlanner 的 7 种倾向

沿用 v2 §六 6.2 不变（红/黄/绿/蓝/紫/金/白 7 种风格 → 决定 AI 优先选哪种颜色 + 图案搭配）。

### 6.3 决策频率（LOD）

| AI 类型 | 视野内 | 视野外 |
|---|---|---|
| 智能 AI（8-10 个） | 60 Hz（每帧） | 2 Hz |
| 轻量 AI（40 个） | 5 Hz | 0.5 Hz |

轻量 AI **不刻纹身**也**不读条**，出生时随机分配一套已刻好的纹身，运行期不变更。

### 6.4 BotBuildPlanner 输入/输出

详见 v2 §六 6.4（v2.1 无变更，BotBuildPlanner 只是在选定三元组后改为发起读条而非瞬时 Equip）。

## 七、风险与开放问题

### 7.1 v2 → v2.1 迁移成本（v2.1 新增）

- **风险**：v2 已交付了 `TattooModule.Equip` + 21 图标 + 336 测试。v2.1 引入读条会不会推翻已实现？
- **结论**：**完全不推翻**。`TattooModule.Equip` 接口零改动——v2.1 只是在调用方（玩家 Tab 键 UI / AI BotBuildPlanner）和 `Equip` 之间**插入一道读条状态机**：
  ```
  [v2]   PlayerUI/Bot ──Equip(p,c,pa)──> TattooModule
  [v2.1] PlayerUI/Bot ──StartReading──> ReadingStateMachine
                                          ├─ 完成 ──Equip──> TattooModule
                                          └─ 中断 ──Cancel──> 扣 50 金币
  ```
- **新增工作量**：
  1. `TattooReadingStateMachine`（新模块，约 200-300 行 C#）
  2. 自纹身界面 UIForm（新 Prefab + ~150 行 UI 脚本）
  3. 读条 UI（屏幕中央 + 角色脚下圆环，~100 行）
  4. `TattooReadingTimeConfig.json` + DataTable 代码生成
  5. AI 端 BotBuildPlanner 改为发起读条而非 Equip（约 20 行调整）
- **不需要改的**：`TattooModule`、21 个 Behavior 策略类、5 张原表、336 测试、CombatHUDForm。
- **取舍**：v2.1 修订只是**加层**，已交付资产 95% 复用。**接受成本**。

### 7.2 读条时长是否过长（脑袋 8s）

- **风险**：脑袋 8 秒在 10-15 min 一局内占比 ~1%，但在战斗节奏中是"两个完整闪避冷却"长度——可能让玩家不敢刻脑袋。
- **缓解**：脑袋的回报是 ScaleFactor=10.0（暴击爆发），是 6 部位最高收益。让"刻脑袋"成为后期 build 决战前的标志性投资（"我准备好了，给我 8 秒"）。
- **监控指标**：playtest 期间记录"脑袋槽刻满率"。若 <30%，下调到 6s。

### 7.3 中断扣 50 金币是否合适

- **风险**：单局基线金币 600-800（参考 09 §五 5.1），50 金币 = 6-8%。过低则中断无痛感，过高则劝退野外刻字。
- **推荐**：保持 50 不变，作为"轻摩擦但不破坏 build 计划"的初始值。playtest 后由 12-数值平衡 GDD 校准。

### 7.4 数值平衡公式（v2 §7.1 沿用）

白色 ColorMultiplier 初始 1.20，由 12-数值平衡 GDD 跑 336 穷举测试统计 DPS 分布，超中位数 +15% 则下调。

### 7.5 配方/颜料局内获取节奏（v2 §7.2 沿用 + v2.1 调整）

- v2 推荐方案：早期 1-2 槽保底 / 中期 3-4 槽 / 后期 5-6 槽。
- v2.1 调整：因初始配方仅 2 个（直线+圆环），早期实际可刻槽位**取决于颜料而非配方**（颜料早期产出 1-3 瓶足够 1-3 槽），配方解锁推动玩家**深度搜图、击杀 Boss、抢拓本**。**节奏不变**。

### 7.6 兽形美术资源压力（v2 §7.4 沿用）

兽形限稀有掉落，美术做 7 色基础 + 通用幻兽剪影。
