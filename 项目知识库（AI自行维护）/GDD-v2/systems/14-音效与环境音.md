## 一、玩家体验目标（30 秒读懂）

**一句话**：音效是手感的隐形组成；环境音承担氛围；关键 NPC 与 Boss 用**伪语**（gibberish）配音点缀"有人/有怪在说话"的临场感，**零本地化成本**。

**核心情感**：
- **手感隐形层** —— 命中音 + 蓄满"叮" + 闪避气流是手感四件套（[02 §2.3 / §5.5](./02-战斗手感.md)）的音频镜像，缺一不可。
- **氛围渗透层** —— 落地 30 秒环境音先于一切提示（[15 §1](./15-世界观与轻剧情.md)），告诉玩家"今天是 AI 废墟 / 异星巢穴 / 病毒沼泽"。
- **决策可读层** —— Boss 入场重低音 + 5-10s 标志性吼叫、阶段切换音、缩圈警报、宝箱"叮"让玩家**不看 UI 也能反应**。
- **角色临场层（v2.1 新增）** —— 纹身师 / 商人 / Boss 的伪语 + 哼声，让 NPC "活着"，但不绑定任何语言。

**反例**：纹身师全程默片只有动作 / Boss 入场无吼叫直接打 / 自纹身 5 秒读条期间一片寂静。

## 二、核心机制 —— 音效分类与触发

### 2.1 五大音效层 + 配音层

| 层 | 用途 | 触发源 | 示例 |
|---|---|---|---|
| **UI 层** | 菜单 / HUD 反馈 | UI 事件 | 按钮 hover / 点击 / 弹窗、Tab 切换 |
| **战斗层** | 攻击 / 受击 / 闪避 | CONTRACT §1.1 / §1.2 | 命中、暴击、蓄满"叮"、闪避气流、死亡 |
| **纹身层** | 装备 / 共鸣 / 升级 / **自纹身 / 附魔** | CONTRACT §1.3 | 纹身刻入、共鸣触发、化身解锁、**刻/钩/烫读条音**、**附魔闪光音** |
| **NPC 层** | 互动 / 交易 / **伪语配音** | CONTRACT §1.5 | 商人吆喝、纹身师工具声、**伪语 1-2 句 + 哼声** |
| **Boss 配音层（新）** | Boss 入场 / 死亡 | CONTRACT §1.7 | **5-10s 标志性吼叫 / 死亡哼鸣** |
| **环境层** | 主题氛围 / 房间气氛 | CONTRACT §1.6 | 主题 ambience loop、风声、机器嗡鸣 |

### 2.2 事件 → 音效触点（v2.1 增补）

| 事件 | 音效行为 |
|---|---|
| `AttackHitEvent` | 武器命中音（按 `WeaponId` 选短刀斩 / 重锤击 / 弓发声） |
| `CritHitEvent` | 暴击重击音 + 金属高频 ping，叠加 Hitstop 4 帧 |
| `ChargedAttackEvent` | 蓄满前 24 帧渐强升调 → 第 24 帧"叮" |
| `DodgePressedEvent` | 闪避气流 swoosh |
| `DamagedEvent` | 受击闷响（玩家叠呼吸急促 stinger） |
| `SkillCastEvent` | 按 SkillId 映射技能音（**仅 2 技能槽**：技能音库从 3 槽配置精简为 2 槽 cooldown 提示） |
| `SkillReadyEvent` | 技能冷却就绪 "叮咚"（2 槽各 1 段不同音高） |
| `EffectAppliedEvent` | DOT / Buff 起手 cue |
| `ActorDiedEvent` | 死亡呼喊（玩家）/ 机械崩溃声（怪）/ Boss 倒地重低音 |
| **`TattooEnchantStartEvent`（新）** | **附魔开始**：纹身师哼声 + 起手"嗡嗡"通电音 |
| **`TattooEnchantTickEvent`（新）** | 附魔过程低频脉动（每秒 1 次） |
| **`TattooEnchantSuccessEvent`（新）** | **词缀附加**短促闪光音（金属高频 + 微 reverb） + 纹身师"嗯！"（伪语满意） |
| **`TattooEnchantFailEvent`（新）** | "嘶——"漏气音 + 纹身师"嗤。"（伪语失望） |
| **`SelfTattooStartEvent`（新）** | **自纹身读条开始**（3-8s）：低沉"刻 / 钩 / 烫"循环音（按部位换 variant） |
| **`SelfTattooCompleteEvent`（新）** | 清脆"叮" + 短暂残响 |
| **`SelfTattooInterruptedEvent`（新）** | "嘶——"中断音 + 玩家闷哼 |
| `BuildChangedEvent` | 纹身装备 confirmation 音（颜色不同微调音高） |
| `CoinChangedEvent` | 金币叮当（Cooldown 防刷屏） |
| `ItemPickedEvent` / `ChestOpenedEvent` | 拾取音 / 宝箱开启重锁解开音 |
| `DeathChestSpawnedEvent` | 低沉 boom |
| **`NPCInteractStartEvent`** | NPC 互动 chime + **伪语开场白 1-2 句**（按 NPC 类型挑 voice bank） |
| **`NPCInteractActionEvent`（新）** | 纹身师"开始/完成/失败"三态各 1-2 句伪语 + 哼声；商人"成交/拒绝"各 1 句 + 哼声 |
| `MapGeneratedEvent` | 主题环境音切换（淡入 2s） |
| `RoomEnteredEvent` | 进房 stinger（普通 / 精英 / Boss 不同） |
| `ZoneShrinkPhaseEvent` | 缩圈警报（每阶段更密） |
| `OutsideZoneTickEvent` | 区外滴答持续低频 tick |
| **`BossSpawnedEvent`** | **Boss 入场重低音 + 5-10s 标志性吼叫（伪语 + 兽性混音）+ BGM 切换** |
| **`BossPhaseChangedEvent`** | BGM 阶段切换（淡入淡出 1.5s）+ **Boss 阶段切换吼**（2-3s 短吼，区别于入场吼） |
| **`BossDiedEvent`** | Boss 倒地重低音 + **死亡哼鸣**（3-5s 低沉收尾，比入场吼弱 -6dB） |
| `RunStartedEvent` / `RunEndedEvent` | 入局 fanfare / 结算音 |
| `BotLODChangedEvent` | AI 进出视野时音效音量渐变（详见 §六） |

### 2.3 环境音随主题切换

`MapGeneratedEvent` 携带 `MapTheme`，AudioModule 订阅后从 `MapThemeConfig.AmbienceCueId`（[15 §4.1](./15-世界观与轻剧情.md)）读取 ambience id，淡入 2s：

| 主题 | Ambience 关键词 | BGM 风格 |
|---|---|---|
| `AI_RUINS` | 电流嗡鸣、伺服转动、警报循环 | 工业电子 / Lo-Fi Synth |
| `ALIEN_HIVE` | 低频脉动、异声呢喃、空间扭曲 | Dark Ambient / Drone |
| `VIRUS_SWAMP` | 蝇虫、气泡、滴液、肺音 | Organic / Body Horror Pad |

## 四、数值与配置

**配置文件**：`Assets/Resources/DataTable/AudioConfig.json`

字段草案（v2.1 增 `VoiceCueId` / `VoiceBankId`）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `AudioId` | string | 音效唯一 ID，如 `sfx_attack_hit_sword` / `bgm_boss_ai` / `amb_ai_factory_loop` |
| `Path` | string | AudioClip 资源路径（走 ResourceModule + ResourceConfig） |
| `Category` | enum | UI / Combat / Tattoo / NPC / **Voice** / Ambience / BGM |
| `Volume` | float | 0.0–1.0 基础音量 |
| `Pitch` | float | 基础音高，默认 1.0；战斗音可微调随机化 ±0.05 |
| `SpatialBlend` | float | 0 = 2D，1 = 3D |
| `MinDistance` / `MaxDistance` | float | 3D 衰减距离 |
| `Cooldown` | float | 同 AudioId 最小重播间隔 |
| `MaxConcurrent` | int | 同 AudioId 并发上限（默认 4） |
| `Variants` | string[] | 同语义多 clip 路径，随机播 |
| `LoopMode` | enum | OneShot / Loop / LoopFade |
| **`VoiceCueId`（新）** | string? | 配音 cue id（如 `voice_tattooer_enchant_start_01`）；非配音音效留空 |
| **`VoiceBankId`（新）** | string? | 所属 voice bank：`tattooer` / `merchant` / `boss_ai` / `boss_alien` / `boss_virus`；用于按 NPC 整体调音量 / 静音 |

**Voice Bank 设计**：
- 每个 NPC 类型 1 个 voice bank，含 3-8 个伪语 clip + 哼声 variants
- 纹身师 bank：`enchant_start_01-02` / `enchant_success_01-02` / `enchant_fail_01-02` / `greet_01-02` + 哼声 4 段
- 商人 bank：`greet_01-02` / `deal_01-02` / `reject_01` + 哼声 3 段
- Boss bank（3 个 Boss 各 1 套）：`intro_roar`（5-10s 标志性）/ `phase_shift_roar`（2-3s）/ `death_groan`（3-5s）

**全局音量分类**：BGM / SFX / Ambience / UI / **Voice**（v2.1 启用，原占位升级为可独立调节 slider）

## 五、UX / UI 触点

### 5.1 BGM 切换节奏（v2.1 单局 10-15min 适配）

> **关键变化**：v2 大局制 25min 缩到 v2.1 单局 10-15min，BGM 切换节奏**整体前移**，避免高潮 BGM 直到末段才响起。

| 时机 | BGM | 时间窗（单局 10-15min 估算） |
|---|---|---|
| 启动 / 主菜单 | `bgm_menu` | — |
| Run 开始过场 | `sfx_run_start_stinger`（1-2s） | T0 |
| **早期：探索曲（轻氛围）** | `bgm_theme_<theme>_explore` | **0:00–3:00** |
| **中期：紧张曲（节奏加速）** | `bgm_theme_<theme>_tense` | **3:00–9:00**（缩圈 Phase 1 切入触发） |
| **后期：决战曲（高强度）** | `bgm_theme_<theme>_climax` | **9:00–15:00**（缩圈 Phase 2 / Boss 临近触发） |
| Boss 战开始 | `bgm_boss_<bossId>`（覆盖决战曲） | Boss 房进入 |
| Boss 阶段切换 | `bgm_boss_<bossId>_phase<n>`（淡入淡出 1.5s） | 阶段事件 |
| Run 结束（胜利） | `sfx_run_victory` + 静默 | — |
| Run 结束（死亡） | `sfx_run_death` + DeathChest stinger | — |

**切换触发器**：由缩圈 phase 事件 + 局内计时双驱动（先到先触发），避免玩家慢节奏卡早期曲、快节奏跳过决战曲。

### 5.2 缩圈警报曲线

| 缩圈阶段 | 警报音节奏 |
|---|---|
| Phase 1 | 每 5s 一声低频警报 |
| Phase 2 | 每 3s 一声 + 心跳 stinger |
| Phase 3 | 每 1s 一声 + 持续低频脉动 |

### 5.3 设置项

- BGM / SFX / Ambience / UI / **Voice** 5 独立 slider
- 静音模式（一键 BGM + Ambience 归 0）
- 听觉无障碍：暴击高亮强制配高频 ping
- **Voice 单独可关**：嫌 Boss 吼吵的玩家可单独静音 Voice，不影响命中反馈

## 六、AI 行为侧需求

**视野内 actor**：音效正常 3D 播放。

**视野外 actor**：
- 高频战斗音 → 不播
- 关键音（死亡 / Boss 入场）→ 保留 -12dB
- **Voice 层（NPC 伪语 / Boss 吼叫）**：仅限**视野内 + 距离 ≤ 30m** 才触发，Boss 入场吼是例外（全图可听，因为 Boss 是局势 cue）

**性能预算**：50 actor 同场，单帧并发音效上限 16 个。**Voice 层独立预算 2 个并发**（避免 5 个 NPC 同时说话糊作一团）。

## 七、风险与开放问题

### 7.1 ⭐ BGM 数量（v2.1 上调）

**推荐**：**8-10 首** —— 主菜单 1 + 3 主题 × 3 段（探索 / 紧张 / 决战）= 9 + Boss 战 1（多 Boss 共用 + 阶段切换淡入淡出）= **10 首**。

**理由**：单局 10-15min 三段式切换是 v2.1 节奏感的核心，每主题只 1 首会显得"打到后半场还是同一首"。10 首仍在 Solo 开发可承受范围（royalty-free 平台 + 主题包可凑齐）。

### 7.2 ⭐ 音效来源

**推荐**：**购买素材包 + 二次合成**，MVP **不自制**。Voice 层走 §7.3。

### 7.3 ⭐ NPC / Boss 配音方式：伪语 vs 完整本地化

**推荐**：**伪语（gibberish）**，明确**不做完整本地化语音**。

| 方案 | 优 | 劣 | 成本估算（5 NPC × 3 态 + 3 Boss × 3 段） |
|---|---|---|---|
| **伪语（推荐）** | 跨语言通用、零本地化负担、风格独特（参考 Diablo 恶魔语 / Sims 模拟语）、Solo 可控 | 玩家无法理解具体语义（但本来就是弱剧情定位） | 1 个配音演员录 1-2 天（即兴）+ 后期混音 1 天 = **1.5–2k RMB 一次性** |
| 完整本地化（中/英/日） | 沉浸感强、剧情可承载 | 3 语言 × 5 NPC × 3 态 × 多句 ≈ 50+ 句 × 3 = 150+ clip，单语 5-8k RMB，三语 ≈ **20-30k RMB**；更新一条台词三语全要返工 | 与 [15 §5.3](./15-世界观与轻剧情.md) "弱剧情、不做配音" 直接冲突 |
| 哼声 only（v2 旧方案） | 最省 | NPC 太呆板，临场感弱 | <500 RMB |

**MVP 策略**：1 个 Solo 友好的配音演员（或 AI 合成 voice 模型如 ElevenLabs 调成伪语模式）录全部 voice bank，混音师做 reverb / EQ 处理使其符合主题（AI 废墟 NPC 加电子失真 / 异星 Boss 加低频脉动）。**所有 Voice 层加 -3 ~ -6dB**，避免压过 SFX 与 BGM。

**伪语写作准则**：辅音爆破 + 鼻音 + 喉音组合，避免任何接近真实语言的音素（不能让英语玩家觉得"像在说英语"或日语玩家觉得"像在说日语"）。参考 *Diablo* demon talk / *Sims* simlish / *Animal Crossing* animalese。

### 7.4 同帧多命中 Hitstop 与音效叠加

`Cooldown=0.03s` + `MaxConcurrent=2`，听觉合并为"一声重击"。

### 7.5 移动端音频压缩与包体

**风险**：v2.1 BGM 从 5 首 → 10 首，Voice bank 新增 ≈ 40-60 clip，包体压力上升。

**缓解**：BGM 用 Vorbis 160kbps（从 192 降）、ambience loop streaming、Voice clip 用 ADPCM 22kHz 单声道（人声不需要立体声）。Voice 总包体目标 ≤ 8MB。详见 13-AudioModule 详设。

### 7.6 自纹身读条音中断处理

读条期间任何受击 → 立刻播 `SelfTattooInterruptedEvent` 中断音 + 强制结束循环音；与 [01](./01-纹身构筑系统.md) 自纹身规则配套。
