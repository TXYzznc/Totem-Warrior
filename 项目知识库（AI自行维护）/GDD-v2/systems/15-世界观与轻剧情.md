## 1. 玩家体验目标

**一句话**：玩家落地 30 秒内，**不读一行文字**也能闻出这一局是"机械废墟"还是"异星巢穴"还是"病毒沼泽"——而且 **第一眼看上去就像一款精致的 2.5D 动作 Roguelike**，是 Hades 路线，不是 PS1 复古路线。

### 1.1 美术风格定位（v2.1 锁定）

| 维度 | 锁定值 | 备注 |
|---|---|---|
| **风格大类** | 似 Hades 的 **精致 2.5D** | 3D 场景几何 + 高质量手绘贴图 + 鲜艳饱和色彩 |
| **视角** | 俯视角斜视镜头（top-down isometric / 3/4 view） | **关键：让玩家纹身在角色身上可见**——这是构筑流派最重要的可视化资产 |
| **角色** | 3D 模型 + 手绘风格贴图 + Hades 风的描边与高光 | 局内立绘走 Hades 路线（2D 半身像 + 高对比光影） |
| **场景** | 3D 模型 blockout + 高质量手绘贴图覆盖；地面图案 / 装饰物有"画"出来的笔触感 | 不追求 PBR 写实，追求"像被画师精修过的 3D" |
| **VFX** | 鲜艳、饱和、轮廓清晰；雷电 / 异变 / 毒云 三系视觉签名截然不同 | 颜色 = 元素 = 主题，三者一致 |
| **UI** | Hades 风羊皮纸 + 金属边框 + 主题强调色微调 | 详见 13 |

### 1.2 玩家身份（v2.1 扩展）

玩家扮演**异能者（Tattooed）**——身体可被纹身改造、激发出元素之力的人。

**背景双兼容**（玩家选角时可二选一，对玩法 0 影响，仅影响开场短动画与 NPC 对话彩蛋）：

- **逃跑实验体**：从某个末日成因的源头（AI 工厂 / 外星实验室 / 病毒研究所之一）逃出，身上的纹身是实验残留
- **游侠 / 拾荒者**：末日后流浪多年，主动寻找纹身师为自己续命/续战的赏金猎人

> 不做"两条主线剧情"——身份只是 flavor，不分支玩法。

### 1.3 30 秒落地体验拆解

- **第 0 秒**：场景配色 + 环境音先于一切提示，告知"今天的世界长什么样"。
- **第 10 秒**：刷出的第一只怪物形态与场景同源——AI 区是失控机器人，外星区是异变触手生物，病毒区是肿胀变形的生物。
- **第 30 秒**：捡到的第一管颜料颜色与主题暗合（AI→黄、外星→紫、病毒→绿），玩家**无须 NPC 解释**就理解"这局适合走雷电 build / 异变 build / 自然 build"。
- **整局过程**：世界观不打断节奏。没有强制剧情过场、没有长台词、没有暂停的对话框。
- **跨局重玩**：每局主题不同 → 视觉记忆点不同 → "上一局是机械废墟，这局想看看外星"成为再来一局的隐藏动机。

**反例**（明确不要）：
- 不要开局长动画讲世界设定。
- 不要 NPC 站着讲 3 段台词玩家才能走。
- 不要"读懂 codex 才能理解 build 思路"。
- **不要**因为美术风格升级就堆细节——单局节奏仍是 Roguelike，不是步行模拟器。

## 2. 核心机制

### 2.1 三大末日主题定义（v2.1 视觉关键词更新到 Hades 风）

| ThemeId | 显示名 | 主导色（Hades 化色板） | 视觉关键词（Hades 精致 2.5D） | 怪物风格 | 环境音关键词 |
|---|---|---|---|---|---|
| `AI_RUINS` | AI 废墟 | 主黄 `#FFD93D` / 银灰 `#9DA3A8` / 电蓝高光 `#3DB4FF` / 暗部冷紫 `#2A2438` | **失控霓虹工厂**：3D 几何走断电摩天楼 / 伺服马达堆 / 断裂电缆 + 手绘贴图加电流故障与黄色警示线；地面有"画"出来的油渍与电路板纹理 | 机械、伺服、电火花、骨架外露，**带 Hades 风金属高光与轮廓描边** | 电流嗡鸣、伺服转动、警报循环 |
| `ALIEN_HIVE` | 外星侵蚀 | 主紫 `#9B59FF` / 暗洋红 `#C13584` / 黑 `#0E0B1A` / 异光青 `#5EEAD4` 点缀 | **非欧巢穴**：3D 几何带非对称扭曲（轻微 vertex displacement）+ 手绘菌丝与触手贴图；地面发出脉动紫光 | 触手、共生、异变、不对称肢体，**Hades 风高对比剪影 + 边缘异光描边** | 低频脉动、异声呢喃、空间扭曲 |
| `VIRUS_SWAMP` | 病毒变异 | 主绿 `#7FD13B` / 黄绿 `#C8E550` / 锈红 `#A0522D` / 暗沼黑绿 `#1A2615` | **感染沼泽**：3D 几何走肿胀植物 / 倒塌建筑被植被吞噬 + 手绘脓液与孢子云贴图；地面湿润反光 | 肿胀、流脓、过度增生、共生体，**Hades 风厚涂质感 + 边缘湿润高光** | 蝇虫、气泡、滴液、肺音 |

**Hades 风共性签名**（三主题共用，保证"看起来像同一款游戏"）：
- 强对比光照（关键光 + 边缘光 + 主题色 rim light）
- 角色与怪物有清晰轮廓描边（防止在复杂场景里看不清）
- 鲜艳饱和的局部色块（不是写实压灰）
- 命中 / 死亡 / 开宝箱等关键瞬间有 Hades 风的 hit-stop + 屏幕震 + 鲜艳粒子爆裂

### 2.2 纹身颜色 ↔ 末日成因映射（与系统 01 共享真理）

| 颜色 | 元素定位 | 主题偏置 | 通用性 |
|---|---|---|---|
| 🟡 Yellow | 雷电 | **AI 废墟**（强偏置） | 任何主题地图均可掉落，AI 主题概率显著上升 |
| 🟣 Purple | 异变 / 虚空 | **外星侵蚀**（强偏置） | 同上 |
| 🟢 Green | 自然 / 毒 | **病毒变异**（强偏置） | 同上 |
| 🔴 Red | 火焰 | 通用（病毒、外星偶发） | 各主题均可掉落 |
| 🔵 Blue | 冰霜 | 通用（外星偶发） | 各主题均可掉落 |
| 🟨 Gold | 神圣 / 古文明 | 通用（任意主题彩蛋点位） | 稀有，与主题无强绑定 |
| ⚪ White | 纯能 / 基础 | 通用 | 起手默认掉落 |

**设计要点**：颜色不是"只能在 X 主题刷"，而是"在 X 主题里掉落概率提升"。这样既让世界观服务于 build 方向，又不强行限制玩家自由。

### 2.3 主题决定的是"地图资源池"，不是"剧情线"

世界观本身**不讲故事**，它讲的是"这一局的素材是什么颜色"：

```
主题选定 (Run 开始)
   │
   ├──> 地形资源池 (07 地图)：场景 prefab、tileset、装饰物
   ├──> 怪物池 (11 怪物)：spawn 表中的 enemyId 权重
   ├──> 事件池 (10 事件)：主题专属随机事件优先采样
   ├──> 颜料掉落偏置 (01 纹身)：颜色权重 + 5~10%
   └──> 环境音 cue (14 音效)：环境层与主题绑定
```

| 下游系统 | 耦合方式 | 数据流 |
|---|---|---|
| **07 地图生成** | 主题决定地形资源池、tileset、装饰物（Hades 风 3D blockout + 手绘贴图） | `MapThemeConfig.TerrainPoolId` → MapGenModule |

**契约要求**（写入 CONTRACT.md）：
- `MapThemeConfig` 由 WorldModule 在 Run 开始时确定并广播 `MapThemeSelectedEvent`，其余模块订阅后从配置表读取自己关心的字段。
- 主题选择算法本身（随机？玩家选择？周常？）由 [07 地图生成 GDD] 定义，本文档只规定"主题一旦选定，所有下游怎么用"。
- v2.1 新增：`CharacterBackgroundSelectedEvent`（角色选择阶段广播），由 14-音效订阅以决定 NPC VO 变体。

### 4.2 颜色偏置量级

- **强偏置颜色**：1.5 ~ 1.8（明显感觉到"这局好像黄墨特别多"）
- **次偏置颜色**：1.1 ~ 1.2（轻微）
- **通用颜色**：1.0
- **抑制颜色**：0.6 ~ 0.8（不希望出现的颜色，例如外星主题压制金色）

**调节原则**：偏置≠垄断。任意主题任意颜色都可能掉，只是概率不同。让玩家有"运气好捡到反流派颜料"的惊喜空间。

## 5. UX / UI 触点

### 5.1 Run 开始：1-2 秒主题揭示动画

- **入场动画**：玩家落地前 1-2 秒，屏幕中央以主题主导色显示一行字幕：「今日主题：AI 废墟」
- **配色微调**：HUD 强调色（生命槽边框、缩圈警告色等）切换为主题强调色，**不改任何功能 UI**。
- **环境音淡入**：在动画结束的同时环境音层无缝接入。
- **v2.1 新增**：异能者背景为「逃跑实验体」时，若当前主题恰好与背景源头一致（如逃自 AI 工厂、本局是 AI 废墟），屏幕角落额外闪一行 flavor text「熟悉的味道...」——0 玩法影响，纯彩蛋。

### 5.2 局内：弱剧情入口（可选挖掘，不可错过的玩家也无所谓）

- **物品 / 装备简短描述**：1-2 行 flavor text（例如颜料瓶上写「来自第 7 号伺服塔的残余电流」），玩家悬停 inventory 可见，但**不主动弹**。
- **NPC 台词 + 轻微配音（v2.1 新增）**：纹身师 / 商人 NPC 各保留 1-2 行环境台词，台词内容根据主题轻微变化（如 AI 主题下纹身师可能说"这块墨是从机器人尸体里榨出来的"）。**v2.1：关键 NPC 加哼声 / 笑声 / 1-2 句轻微配音**，强化 Hades 风的"NPC 有生命感"。详见 14-音效 v2.1。
- **环境叙事点位**：地图随机生成 0-2 个特殊点位（破碎实验台、外星遗物、感染者尸堆），靠近时无 UI 提示，**只用视觉本身讲故事**。在 Hades 精致 2.5D 风下，这些点位本身就是高质量手绘资产，玩家会主动驻足看两眼。

### 5.3 不做

- ❌ 不做 codex / lore book / 收藏图鉴系统（与"弱剧情"哲学冲突）。
- ❌ 不做**全 NPC 全程配音**（成本与轻剧情定位不符）；v2.1 只做**关键 NPC 极少量**的哼声 + 1-2 句 VO。
- ❌ 不做强制 NPC 对话（玩家可走开，对话只是环境点缀）。
- ❌ 不做**全角色背景剧情线**（异能者身份只是 flavor，不分支玩法）。

## 6. AI 行为侧需求

**对 AI 玩家（BotControllerModule）**：保持简洁，AI **对主题无认知**。

- AI 不会"看到 AI 废墟所以想刻黄色 build"。
- AI 装备纹身的逻辑由 `BotBuildPlanner` 根据预设 build 模板 + 当前已捡到的颜料决定。
- 主题对 AI 的影响是**间接的**：因为颜料掉落偏置，AI 在 AI 主题地图上自然就更容易捡到黄墨 → 自然就更可能走雷电 build。

**未来扩展位**（不在本期）：
- 可加"主题专属 AI build 倾向"配置：`BotBuildTemplate.ThemePreference`，让某些 AI 模板在特定主题下被优先选取（例如外星主题下倾向选"异变流"AI 模板）。
- 本期不实现，但 BotBuildPlanner 的接口设计要预留 `currentTheme` 参数，避免未来加扩展时改动通信。

## 7. 风险与开放问题（v2.1 重点重写）

### 7.1 风险评估

| 风险 | 等级 | 应对 |
|---|---|---|
| **Hades 风精致 2.5D 在 Solo + 1 人 AI 工具产能下是否可行** | 🔴 高（v2.1 核心新增） | **降级方案与产能假设详见 7.2** |
| **三大主题美术成本（3 套场景 + 3 套怪物 + 3 套 VFX）** | 🔴 高 | 三主题共享"末日改造痕迹 + Hades 风轮廓描边 + 强对比光"美术语言；3D 模型走 blockout 复用，仅替换贴图色板与少量主题道具，大幅复用资源 |
| **主题切换导致玩家"踩雷"** | 🟡 中 | 玩家想刻黄色 build 但今天是病毒主题怎么办？→ 不强迫单局完成 build，允许跨局规划。颜色仍能掉，只是偏置低 |
| **弱剧情会不会显得"没世界观"** | 🟡 中 | 通过环境叙事 + flavor text + 主题揭示动画 + v2.1 关键 NPC 轻微配音的组合，让世界感"渗"出来。Hades 精致 2.5D 视觉本身就是世界观的最强承载 |
| **HUD 配色变动影响可读性** | 🟢 低 | 只动强调色不动功能色（伤害数字红 / 治疗绿等保持不变），交付前与 art-ui 复核对比度 |
| **关键 NPC 配音的语言 / 演员 / 一致性** | 🟡 中 | 优先用 AI 语音合成（如 ElevenLabs）做 placeholder + 上线前关键 NPC 走 1-2 小时真人录音；如成本过高，降级为纯哼声 / 笑声 / 叹气等非语言 VO |

### 7.2 Hades 精致 2.5D 可行性诚实评估（v2.1 核心）

**产能假设**：1 人 Solo + AI 工具（绘图模型、ImageCut、Blender 自动化、game-asset-mcp 等）。

**是否可行？— 结论：可行，但必须分级处理。**

| 资产档次 | 占比 | 制作方式 | 工期权重 |
|---|---|---|---|
| **S 级：玩家会反复盯着看的资产** | ~5% | **关键 Boss 立绘 / 主菜单背景 / 角色选择立绘 / 开场动画 keyframe** — AI 出图 + **手工精修** | 高（单件耗时） |
| **A 级：玩家会注意但不细看的资产** | ~25% | **场景 3D blockout + Hades 风手绘贴图 + 怪物 3D 模型** — AI 草图 + Blender + AI 贴图工具，少量手工修边 | 中 |
| **B 级：占位但视觉一致** | ~70% | **道具图标 / 普通怪物贴图变体 / 装饰物 / VFX 变体** — AI 出图 + 自动处理（rembg / ImageCut / 批量风格化）+ 不修 | 低（批量产出） |

**核心策略**：
1. **MVP 期用素材包 + AI 出图作为底盘**（场景 prefab 从 Synty / Quaternius / Itch 找 Hades 风近似包；纹理与 VFX 用 AI 批量生成）
2. **关键资产手工精修**：S 级资产逐件磨，每件容许 4~16 小时
3. **风格一致性靠 art bible + AI 提示词模板兜底**——所有 AI 出图都走 ai-art SKILL，统一的"Hades 风 / 主题色板 / 描边规范"提示词后缀，保证视觉收敛
4. **持续降级阀值**：若 6 个月里程碑时美术工时超预算 30%，触发 **Plan B**：
   - 三主题缩减为两主题（先 AI + 外星，病毒后置）
   - 怪物变体从每主题 6~8 种缩为 4~5 种
   - 怪物 3D 模型改为高质量 2D sprite + 3D 场景（伪 2.5D），削减建模工时
5. **持续降级阀值 2**：若 Plan B 仍超预算，触发 **Plan C**：放弃 3D 场景，全 2D Hades 风手绘 + Spine 动画。视觉档次不变，仅维度降一档。

**底线**：**风格一致性 > 资产数量 > 资产精度**。宁可 50 个统一 Hades 风的占位资产，也不要 100 个风格分裂的精修资产。

### 7.3 开放问题（需要后续闭环）

1. **是否需要 codex / NPC 长台词系统？**
   - **推荐结论**：**不做**。仅保留 UI 字幕（开局主题揭示） + NPC 1-2 行短台词 + 物品 flavor text + v2.1 关键 NPC 轻微配音。不做对话树、不做 codex。
   - **依据**：弱剧情定位 + Solo 开发成本 + 不打断 BR 节奏。
   - **若未来扩展**：可加"角色解锁背景故事"短篇（一次性阅读，不影响局内）。

2. **主题选择算法**（随机？玩家选择？周常？） → 交给 [07 地图生成 GDD] 决定，本文档不裁定。
   - 倾向建议：MVP 期默认"完全随机"；后续可加"每日固定主题"提供刷主题动机。

3. **混合主题地图**（例如一张图同时有 AI 区与外星区） → v1 草案有提，本期 GDD **不做**，未来可作为"混乱主题"扩展位预留。

4. **异能者背景的具体开场动画长度** → 暂定 5~8 秒短动画，可跳过；具体由 [06 角色设定 GDD] 决定。

5. **关键 NPC 配音的语言策略** → 中文 first？还是 Hades 那种"虚构语言哼声 + 字幕翻译"？倾向后者（成本低、可避免英文配音的违和）。最终交给 [14 音效 GDD] v2.1 裁定。

> **设计哲学回应"为什么这样而不是那样"**：
>
> 玩家想要的是"末日废墟的代入感"而不是"末日剧情的理解感"。前者靠**视觉 + 声音 + 资源池**就能达成，后者要靠文字与配音——而文字与配音会**打断 BR 节奏**。所以本作选择"用素材讲世界观"，让玩家**不读字也懂氛围**，读字了能挖到额外的小彩蛋。
>
> v2.1 进一步锁定 **Hades 精致 2.5D** 路线：俯视角斜视镜头让玩家纹身随时可见，3D 场景 + 高质量手绘贴图给末日世界以"被画师精修过"的质感，鲜艳饱和的色板让三大主题各自难忘。这是 Hades / Dead Cells 路线，不是 Witcher / Mass Effect 路线，也不是 PS1 复古 / 像素 Roguelike 路线。
>
> 1 人产能能不能做？**能**——只要 70% 走 AI 批量 + 占位、25% 走 AI + 少量精修、5% 真正手工磨；只要风格一致性兜底；只要降级方案随时可启动。
