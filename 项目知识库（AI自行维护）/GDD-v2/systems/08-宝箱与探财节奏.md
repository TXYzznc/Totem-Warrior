## 一、玩家体验目标

**核心驱动力**："再走一段" 循环——玩家在局内每 2–3 分钟应遇到至少 1 个有意义的拾取节点，保证探索欲望持续燃烧而不枯竭。

| 体验层次 | 描述 |
|---|---|
| **即时满足** | 开箱画面 + 飘字给予即时的感官反馈；掉落物带颜色高亮区分稀有度 |
| **策略驱动** | 稀有颜料 / 配方直接影响 build 走向，促使玩家在"抢箱"与"安全苟圈"之间决策 |
| **复仇与贪婪** | 死亡宝箱是局内最高价值战利品，驱动幸存者绕路"收尸"，形成短暂的局部热点冲突 |

**10–15 分钟局内拾取节奏校验**（对齐 12-数值平衡与曲线 §2.7 tattooSlotsByTime，以 12.5min 中值为基准）：

| 阶段 | 时间 | 期望累计颜料瓶 | 主要来源 |
|---|---|---|---|
| 早期 | 0–3 min | 1–2 瓶 | 出生区普通箱 + 零散怪掉落 |
| 中期 | 3–8 min | 3–6 瓶 | 精品箱 + 精英怪 + 小商人 |
| 后期 | 8–15 min | 7–12 瓶 | Boss 箱 + 死亡宝箱 + 缩圈外的遗弃箱 |

颜料消耗量（刻 1 个纹身 = 1 瓶颜料）与上表对应，确保 t=12.5 时刻满 4–5 槽位的中位数玩家恰好消耗 4–5 瓶，剩余颜料为备用缓冲。

## 二、核心机制

### 2.1 四种宝箱类型

| 宝箱类型 | ChestTypeId | 开启耗时 | 地图数量/局（v2.1） | 颜色 / 视觉标识 |
|---|---|---|---|---|
| **普通箱** | `Normal` | 即时 | 12 个 | 灰色木箱 |
| **精品箱** | `Premium` | 1.5 s 动画 | 6 个 | 蓝光金属箱 |
| **Boss 箱** | `Boss` | 2.0 s 动画 | 2 个（Boss 必掉） | 紫金发光 |
| **死亡宝箱** | `DeathChest` | 0.8 s 动画 | 动态（每个玩家死亡生成 1 个） | 红色骷髅标记 |

总箱子密度：每局约 **20 个固定箱**（Normal + Premium + Boss），适配 10–15min 短局节奏（约 1.6 箱/min）。

### 2.2 掉落物类型与分布

四类可掉落物：

1. **颜料瓶**（InkBottle）：驱动纹身刻制，核心资源；分三档稀有度（详见 §2.3）
2. **配方**（Recipe）：解锁特定纹身图案，四条来源（详见 §2.4）
3. **武器 / 技能道具**（Equipment）：即拾即用，提升即时战斗力
4. **金币**（Coin）：用于商人购买 / 纹身重刻，软性货币

### 2.3 颜料三档

| 档位 | 颜色 | 稀有度 ID | 主要来源 | 飘字颜色 |
|---|---|---|---|---|
| **常见** | 红 / 黄 / 绿 / 蓝 | `Common` | 地图杂兵、普通箱 | 对应颜色本色 |
| **稀有** | 紫 / 金 | `Rare` | 精英怪 / Boss / 高级宝箱 / 商人售卖 | 紫 `#CC88FF` / 金 `#FFD700` |
| **传说** | 白 | `Legendary` | Boss / 隐藏宝箱 | 白 `#FFFFFF`（描边黑） |

稀有度对纹身效果有加成（详见 01-纹身构筑系统），同颜色高档颜料可替代低档颜料刻制，但消耗 1:1 等量。

### 2.4 配方来源（四条路径）

| 来源 | 获取方式 | 持久性 |
|---|---|---|
| **局外解锁** | 成就达成 / 累计 Run 数解锁，进入全局配方池 | 永久 |
| **Boss 掉落** | Boss 死亡必掉 1–2 张完整配方（走 Boss LootTable）| 本局携带，局后存档 |
| **宝箱 + 事件三选一** | 精品箱 / 特殊事件节点提供 3 张候选配方供玩家选 1 | 本局携带，局后存档 |
| **死亡玩家拓本** | 继承死亡玩家已刻纹身对应配方的一半（向下取整），**仅本局可用** | 本局临时；局结束清除 |

> 拓本机制说明：死者身上刻有 4 种配方 → 拓本数 = floor(4/2) = 2 张，随机从 4 张中抽取 2 张写入拾取者临时配方槽；局结束时临时槽清空，不写入全局配方池。客户端实现时使用独立的 `TempRecipeSlot` 字段与 `IsTemporary` flag 标识。

### 2.5 死亡宝箱内容规则（v2.1 新规）

死亡宝箱不走 LootTable 随机，内容由 `DeathChestRuleConfig.json`（§4.3）驱动，按以下规则从死亡玩家快照计算：

| 物品类 | 继承量 | 说明 |
|---|---|---|
| **颜料瓶** | floor(持有量 / 2) | 向下取整；另一半消散，模拟"战场散逸" |
| **配方拓本** | floor(已刻纹身配方数 / 2) | 向下取整，随机抽取；**仅本局可用**（`IsTemporary = true`） |
| **金币** | floor(持有金币 × 0.5) | 向下取整；另 50% 消散 |
| **武器 / 技能道具** | 100% | 已装备的武器也一并放入宝箱 |
| **纹身本身** | 不脱落 | 纹身是身体改造，不可被抢夺；杀手获得的是颜料+配方，而非直接继承 build |

> **设计意图**：颜料减半（v2.0 全量 → v2.1 半量）是为了降低单次击杀的资源暴利，使颜料经济不因高频 PVP 而严重通胀；配方拓本以"仅本局可用"的方式补偿了颜料损失，同时不破坏全局配方池的稀缺性；武器 100% 继承维持战力转移的即时冲击感；金币 50% 沿用 v2.0 对称直觉数字。

**balance-check 结论（v2.1 死亡宝箱）**：

| 检测项 | 预期 | v2.1 数值 | 状态 |
|---|---|---|---|
| 击杀激励 vs 苟圈 | 击杀 EV > 同等时间 3 个普通箱 | 颜料 floor(N/2) + 武器全量 + 拓本 ≈ 远超普通箱 | 健康 |
| 单次颜料暴利风险 | 不超过 t 时刻普通路径 2 倍产出 | t=10 期望持有 5 瓶 → 击杀得 floor(5/2)=2 瓶，不超过 2 倍 | 健康 |
| 拓本退化（永久配方外泄）| 拓本本局有效，不影响全局池 | `IsTemporary` flag 隔离 | 已覆盖 |
| 苟圈策略（不参与 PVP）| 需负收益差距 | 拓本 + 武器只有击杀才能获得，苟圈无此收益 | 已覆盖 |

## 三、与其它系统的耦合

| 系统 | 耦合内容 | 数据流方向 |
|---|---|---|
| **07-地图生成** | MapGenModule 按 ChestConfig.SpawnWeight 在房间节点放置宝箱；保证每个区域至少 1 个 Normal 箱 | 07 → 08 |
| **09-纹身师与商人NPC** | 商人库存颜料（含三档）/ 配方通过 EconomyModule 共用 ItemId 定义；商人补货时参考局内已流通配方稀缺度 | 08 ↔ 09 |
| **11-怪物与Boss** | 精英怪死亡掉落走 LootTable（TableId: EliteEnemy）；Boss 死亡强制生成 1 个 Boss 箱 | 11 → 08 |
| **经济模块（EconomyModule）** | 所有金币进出经过 `CoinChangedEvent`；拾取颜料/道具经过 `ItemPickedEvent` | 08 发布事件 → EconomyModule 处理 |
| **01-纹身构筑系统** | 颜料瓶 = 刻纹身的消耗品，InkBottle 的 ItemId 对应 TattooColorConfig 中的 ColorId + RarityTier | 08 产出颜料 → 01 消耗 |
| **16-BotControllerModule** | Smart Bot 监听 `DeathChestSpawnedEvent`，记录死亡宝箱坐标，列为优先探索目标 | CONTRACT §1.4 |

**CONTRACT 事件引用**（来自 `05-gdd-v2-full-design-docs/CONTRACT.md §1.4`）：

```csharp
// 开箱时发布
ChestOpenedEvent  { Actor Opener; ChestType Type; LootRollResult Roll; }

// 玩家死亡时由 EconomyModule 生成死亡宝箱并发布
DeathChestSpawnedEvent { Actor DeadActor; Vector3 Pos; List<int> ItemIds; List<int> TempRecipeIds; }

// 拾取死亡宝箱内容时发布
DeathChestLootedEvent  { Actor Looter; Actor DeadActor; }
```

> v2.1 变更：`DeathChestSpawnedEvent` 新增 `TempRecipeIds` 字段，携带本局临时配方 ID 列表（`IsTemporary = true`）；EconomyModule 写入临时配方槽时需同步标记。

## 五、UX / UI 触点

### 5.1 宝箱开启动画

- **普通箱**：无延迟直接弹出掉落物（保证探索流畅不卡顿）
- **精品/Boss 箱**：1.5–2.0 s 的开盖动画（光效由 VFXModule 触发 `VFXTriggerEvent`）；动画期间玩家进入"开箱蹲姿"，不可移动也不可攻击（制造开箱风险感）
- **死亡宝箱**：短 0.8 s 动画 + 音效（哀鸣声），随后掉落物以爆发式粒子弹出；若含拓本，弹出额外"拓本获得"飘字

### 5.2 掉落物飘字规范

| 物品类型 | 飘字颜色 | 格式 | 说明 |
|---|---|---|---|
| 颜料瓶（常见：红/黄/绿/蓝）| 对应颜色本色 | `+颜料 ×N` | N = 掉落数量 |
| 颜料瓶（稀有：紫/金）| 紫 `#CC88FF` / 金 `#FFD700` | `+稀有颜料！` | 加 `！` 强调 |
| 颜料瓶（传说：白）| 白 `#FFFFFF`（描边黑）| `+传说颜料！！` | 双 `！` + Toast 弹窗 |
| 配方碎片 | 橙 `#FF9900` | `配方碎片 N/3` | 提示凑齐进度 |
| 完整配方 | 橙 `#FF9900`，加粗 | `配方解锁！` | 弹出 Toast |
| 配方拓本（临时）| 青 `#00CCFF`，斜体 | `[拓本] 配方名（本局）` | 标注 "本局" 提示临时性 |
| 武器/道具 | 对应稀有度色（白/绿/蓝/紫/金）| 道具名 | 与装备面板色系一致 |
| 金币 | 黄 `#FFD700` | `+N coin` | 小字，不喧宾夺主 |

### 5.3 死亡宝箱地图标记

- 死亡宝箱生成时，小地图上立即出现**红色骷髅图标**（高亮诱饵）
- 图标持续显示，直至宝箱被拾取（`DeathChestLootedEvent` 后消失）
- 缩圈后死亡宝箱若在圈外，图标变为**灰色骷髅**（提示风险），不消失但优先级降低
- 死亡宝箱坐标向局内全体玩家广播（含 Bot），制造热点争夺

## 六、AI 行为侧需求

Smart Bot（BotControllerModule）在探财行为上需满足：

1. **死亡宝箱为最高优先级探索目标**（`BotPriority = 10`）：监听 `DeathChestSpawnedEvent`，将宝箱位置压入目标队列头部；权重高于圈内普通探索和 PVP 交战
2. **记忆已知宝箱位置**：经过附近（≤12 m）但未开的箱子记入"待开列表"，地图探索完成度影响 Bot 的路径规划效率
3. **开箱安全性评估**：Bot 开精品箱 / Boss 箱前检查周围 20 m 内是否有玩家；若有玩家则退出，等待玩家离开或战斗结束后再开
4. **死亡宝箱竞争**：多个 Bot 同时感知同一死亡宝箱时，距离最近的 Bot 优先；其余 Bot 降低该目标权重，转向次优目标
5. **拓本价值评估**：Bot 拾取死亡宝箱时，若获得拓本配方与当前 build 颜料储量匹配，立即触发纹身决策逻辑（优先级提升）

## 七、风险与开放问题

### 7.1 死亡宝箱内容——v2.1 已定策略

| 争议项 | v2.1 决策 | 理由 |
|---|---|---|
| 颜料继承比例 | **floor(N/2)** | 降低单次击杀颜料暴利，防止资源通胀；拓本补偿了部分损失 |
| 配方拓本是否继承？ | **继承 floor(已刻数/2)，仅本局有效** | 给击杀以"build 预览"奖励而不永久外泄配方稀缺性 |
| 武器/技能道具是否继承？ | **100% 全继承** | 维持战力转移的即时冲击感；武器是核心击杀激励 |
| 纹身是否脱落？ | **不脱落** | 纹身是身体改造，脱落会破坏 build 所有权感 |
| 金币继承比例？ | **50%** | 对称直觉数字，易理解；平衡"击杀价值"与"弱者拾取性价比" |

### 7.2 宝箱密度（v2.1 调整）

- v2.0：30 个/局（25min 设计），目标 1.2 箱/min
- v2.1：**20 个/局（10–15min 设计）**，目标 1.6 箱/min，密度适度提升符合短局更快节奏

| 密度风险 | 阈值 | v2.1 状态 |
|---|---|---|
| 过密（资源通胀） | > 30 个/局 | 20 个，安全 |
| 过稀（早期挫败感）| < 12 个/局 | 12 Normal + 6 Premium + 2 Boss = 20，安全 |

### 7.3 开放问题

- **稀有颜料的颜色权重**：死亡宝箱中继承的颜料颜色是否影响 meta（某颜色系统性稀缺）？建议 playtesting 后决定是否引入"局内偏色"机制。
- **传说颜料（白）的独占性**：是否仅隐藏宝箱出现？当前配置 Boss 箱 25% 也可掉落，推荐保留以确保中位数玩家在约 3–5 局内至少见到 1 次。
- **Boss 独占配方**：是否有仅在 Boss 箱出现的专属图案配方（形成探图目标感）？推迟至 v1.1 讨论。
