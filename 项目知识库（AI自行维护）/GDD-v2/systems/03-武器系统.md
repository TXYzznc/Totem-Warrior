# 03-武器系统

> **主导 Agent**：gd-system
> **协作 Agent**：gd-lead（体验目标审定）/ client-unity（DataTable 落地）/ art-2d（武器图标）
> **依赖系统**：02-战斗手感（帧表基线 + AttackHitEvent）/ 12-数值平衡与曲线（BaseDamage 基线 / scaleStat 定义）/ 06-角色设定与骨架（角色尺寸影响武器 hitbox 范围）
> **被依赖系统**：01-纹身构筑系统（右臂 scaleStat = 武器 BaseDamage）/ 04-主动技能（武器类型影响技能释放动画）/ 08-宝箱与探财（武器掉落表）/ 09-纹身师 NPC / 11-怪物与 Boss（怪物使用同一武器配置）/ 16-BotControllerModule（AI 武器选择）

## 一、玩家体验目标

**一句话**：拾起一把武器的那一刻，玩家应立即感受到"这把武器是我这局的基调"——近战刀让人想贴身连刀，重锤让人期待那一下蓄满，枪让人想和距离玩游戏。

**核心情感目标**：

- **武器决定普攻基调**：武器不是数值上的"大数字替换小数字"，而是改变了玩家发动攻击的节奏和欲望。短刀手感轻快，蓄力倍率低但频率高；重锤频率低，但蓄满那一击的 Hitstop + 伤害应有强烈的重量感反馈。
- **与纹身解耦但联动**：武器和纹身是两条独立的成长轴。玩家可以先拿一把武器再刻纹身，也可以先刻纹身再找匹配武器。两者联动体现在：武器的 `BaseDamage` 作为右臂纹身 `scaleStat` 的基线——武器越强，纹身伤害的绝对值越高（但倍率不变），这是机制上的正向联动，不是强制绑定。
- **武器切换有机会成本**：玩家最多持有 3 把武器，1-3 数字键切换。切换本身不带动画延迟（即时切换），但新武器的第一击有 Startup 帧，玩家在 PVP 中感受到的是"换了把，得重新找节奏"。

**反例对照**：

- 武器只是不同数字的同一根棍子 → 玩家没有换武器的欲望，build 同质化
- 武器和纹身强绑定（某武器只能配某纹身）→ 降低了策略自由度，违反 GDD-v2 共识
- 武器满地图都是 → 选择困难，拾取无意义感

## 二、核心机制

### 2.1 武器三大类与 MVP 5 把武器

MVP 期共设计 5 把武器，覆盖三大类：

| 类别 | 武器 | 代号 | 设计意图 |
|---|---|---|---|
| **近战** | 短刀 | `sword_basic` | baseline 武器，02-战斗手感帧表基准就以此为短刀计算 |
| **近战** | 重锤 | `hammer_heavy` | 低频高伤，蓄力倍率 2.0×（覆盖默认 1.5×），攻速 -0.2 |
| **远程** | 手枪 | `pistol_basic` | 有限弹药，投射物直线飞行，近距离高频，中远程有效 |
| **远程** | 弓 | `bow_charge` | 必须蓄力才能发射（不蓄力无 Active hitbox），蓄力倍率 1.8×，射程最远 |
| **特殊** | 能量拳 | `fist_energy` | 近战拳击，无弹药，攻速 +0.2，BaseDamage 最低但触发频率最高（利于频繁触发右臂纹身） |

**MVP 外（v1.1 候补）**：投掷武器（手雷/飞镖）、双刀、大剑、步枪。不在本期设计，暂不定义。

### 2.2 武器对帧表三参数的影响

02-战斗手感 §2.2 定义了帧表公式：

```
StartupFrames  = BaseStartup  × (1 - AttackSpeedBonus)
ActiveFrames   = BaseActive   × (1 - AttackSpeedBonus × 0.5)
RecoveryFrames = BaseRecovery × (1 - AttackSpeedBonus)
```

武器通过 `WeaponConfig.json` 中的 `BaseStartup / BaseActive / BaseRecovery / AttackSpeedModifier` 字段覆盖短刀基线：

| 武器 | BaseStartup | BaseActive | BaseRecovery | AttackSpeedModifier | 蓄力倍率 ChargedMul |
|---|---|---|---|---|---|
| 短刀（基线） | 4 帧 | 3 帧 | 8 帧 | 0 | 1.5× |
| 重锤 | 10 帧 | 5 帧 | 18 帧 | -0.20 | 2.0× |
| 手枪 | 3 帧 | 1 帧 | 6 帧 | +0.15 | — （无蓄力） |
| 弓 | — （无普攻直射） | 4 帧 | 12 帧 | 0 | 1.8× （必须蓄满才有 hitbox） |
| 能量拳 | 3 帧 | 4 帧 | 5 帧 | +0.20 | 1.2× |

> 弓的特殊规则：`BaseActive = 0`（普攻即松弦，未蓄力时 hitbox 不生效），玩家需蓄力至 `ChargedReadyThreshold`（24 帧）才能打出有效 Active 帧。这是弓的核心机制差异，由 CombatModule 读 `WeaponConfig.RequiresCharge = true` 判断。

**攻速归一化**：`AttackSpeedBonus` 最终值 = `WeaponConfig.AttackSpeedModifier + TattooPassiveStats.AttackSpeed`，上限 0.6（来自 12-数值平衡 §二 BalanceConstantsConfig.AttackSpeedBonusCap）。

### 2.3 远程武器投射物系统

远程武器命中不走近战 hitbox 接触判定，而是：

1. 武器 `AttackHitEvent` 触发时，CombatModule 读 `WeaponConfig.ProjectileId`
2. 若 `ProjectileId != null`，生成 `ProjectilePrefab`（路径 `Resources/Prefabs/Projectiles/<ProjectileId>.prefab`）
3. 投射物飞行期间持有 `Actor` 信息（发起者 ID），命中时触发 `AttackHitEvent`（与近战共用契约）

投射物关键参数（存于 `ProjectileConfig.json`，本 GDD 定义字段草案）：

| 字段 | 说明 |
|---|---|
| `ProjectileId` | 唯一 ID（与 WeaponConfig 外键关联） |
| `Speed` | 飞行速度 (m/s) |
| `MaxRange` | 最大飞行距离，超出后销毁 (m) |
| `Piercing` | 是否穿透目标（布尔；弓蓄满后可设为 true） |
| `AoeRadius` | 爆炸/范围伤害半径，0 = 单体 (m) |
| `VisualPrefabId` | 投射物视觉 Prefab 路径 |

**弹药系统**：近战武器无弹药限制（`AmmoType = None`）；远程武器有限弹药，消耗完毕后切换至近战副武器或徒手状态（输出降低 60%）。具体弹药数见 `WeaponConfig.json` 的 `MaxAmmo` 字段。

### 2.4 武器稀有度与掉落等级

武器分 4 级稀有度，对应 `Rarity` 字段：

| 稀有度 | 代码值 | BaseDamage 范围 | 掉落来源 |
|---|---|---|---|
| 普通（白） | `Common` | 10–20 | 地图散落，开局密集 |
| 稀有（蓝） | `Rare` | 22–35 | 地图宝箱，怪物掉落 |
| 精良（紫） | `Epic` | 38–55 | 死亡宝箱，商人购买 |
| 传说（金） | `Legendary` | 58–80 | 死亡宝箱（稀有）、圈内高风险区域 |

> BaseDamage 范围来自 12-数值平衡 §三武器引用（白色 10–20，金色 40–60）。本表将精良和传说拆分，金色上限延伸至 80，需在 balance-check 中对 t=18 PVP 一击击杀风险做验证（详见 §七 风险 7.1）。

### 2.5 武器与 AttackHitEvent

每次攻击命中，CombatModule 触发（CONTRACT §1.1）：

```
AttackHitEvent {
    Actor:        发起者 ID
    Target:       受击者 ID
    BaseDamage:   effectMagnitude（由 WeaponConfig.BaseDamage 作为 scaleStat 计入）
    IsCrit:       bool
    IsCharged:    bool
    WeaponId:     string  （新增字段，供 AI 武器切换决策使用）
    ProjectileId: string? （远程时非空）
}
```

> `WeaponId` 字段需同步至 CONTRACT.md §1.1，由 client-unity 在 CombatModule 实装时同步写入。

## 三、与其它系统的耦合

| 系统 | 耦合方式 | 方向 |
|---|---|---|
| **02-战斗手感** | 武器提供 `BaseStartup / BaseActive / BaseRecovery / AttackSpeedModifier / ChargedMul`，帧表公式在 02 定义，武器参数在本 GDD 定义 | 武器 → 战斗 |
| **01-纹身构筑系统** | 右臂纹身的 `scaleStat` = 触发时读 `WeaponModule.CurrentWeapon.BaseDamage`（实时值），武器越强纹身绝对伤害越高 | 武器 → 纹身 |
| **04-主动技能** | 武器类型（近战/远程/特殊）影响技能动画层（Animator Override Controller）；远程武器使用技能时是否消耗弹药由各技能 `ConsumeAmmo` 字段单独控制 | 武器 → 技能 |
| **08-宝箱与探财** | 宝箱掉落表引用 `WeaponConfig.Rarity` + `WeaponConfig.DropWeight` 字段；各稀有度武器掉率见 08-GDD 专项定义 | 武器配置 → 宝箱 |
| **09-商人 NPC** | 商人库存抽取走 `WeaponConfig.TradeEnabled` 字段；价格公式 `price = BaseDamage × 4 × rarityPriceMul`，rarityPriceMul: Common=1.0 / Rare=1.8 / Epic=3.0 / Legendary=5.5 | 武器配置 → 商人 |
| **11-怪物与 Boss** | 怪物和 Boss 携带武器也读 `WeaponConfig`，使用同一帧表规范——Boss 可持传说级武器，普通怪持普通/稀有级 | 武器 → 怪物 |
| **16-BotControllerModule** | Smart AI 主动捡武器并按决策逻辑切换（见 §六）；Light AI 使用出生时随机分配的预设武器，不在运行期切换 | 武器 → AI |

### 4.3 平衡检查——关键数值验算

基于 12-数值平衡 §2.4 的 `enemyHP(t)` 和 `playerEffHP(t)`，对 5 把 MVP 武器做击杀时间（TTK）预估：

**公式回顾**：
```
effectMagnitude = BaseDamage × scaleFactor(右臂=0.8) × colorMul × patternMul × synergyMul
damageDealt = effectMagnitude × elementMul × (1 - resist)
```

**简化验算**（scaleFactor=0.8，colorMul=1.0，patternMul=1.0，synergyMul=1.0，elementMul=1.0，resist=0）：

| 武器 | BaseDamage | 每击伤害（简化） | 普攻频率（基线） | 怪物HP(t=8)=49 理论击杀击数 | 怪物HP(t=18)=95 理论击杀击数 |
|---|---|---|---|---|---|
| 短刀 | 15 | 12.0 | 4次/s | 5击（~1.3s） | 8击（~2.0s） |
| 重锤 | 28 | 22.4 | ~1.2次/s（攻速-0.2） | 3击（~2.5s） | 5击（~4.2s） |
| 手枪 | 12 | 9.6 | ~4.6次/s | 6击（~1.3s） | 10击（~2.2s） |
| 弓（蓄力） | 22×1.8 | 31.7 | ~0.5次/s（蓄力必须） | 2击（~4s） | 3击（~6s） |
| 能量拳 | 8 | 6.4 | ~4.8次/s | 8击（~1.7s） | 15击（~3.1s） |

**设计检验**（来自 12-数值平衡 §2.4）：`playerDPS → enemyHP 击杀时间应在 2.5–5.0 秒`。

- 早期（t=8）：短刀/手枪偏快（1.3s），近边界；弓和重锤符合区间
- 后期（t=18）：大多武器符合；弓后期已偏长（6s），但弓的蓄力穿透 + 纹身联动可补偿

**结论**：短刀/手枪早期击杀偏快，需在 balance-check 迭代中确认是否引起早期资源积累速度过快。推荐保留观察而非立即调整，待 playtest 后以实测 DPS 为准。

## 五、UX / UI 触点

### 5.1 武器槽位 HUD

- 屏幕底部，技能槽右侧，显示 3 个武器卡槽（1 / 2 / 3 数字键切换）
- 当前装备武器：边框高亮 + 名称显示
- 未装备槽位：显示图标（灰色），若为空则显示"—"
- 弹药武器：槽位右下角显示「当前弹药 / 最大弹药」，格式：`12/18`；无弹药显示为红色

### 5.2 武器拾取 / 切换交互

- 地面武器显示悬浮图标（武器名 + 稀有度色框），靠近后显示 `[E] 拾取`
- 若武器槽已满（3/3），拾取会弹出"替换哪把？"选择面板（槽 1 / 槽 2 / 槽 3 / 取消）
- 替换过程中当前武器不被强制卸下，玩家取消仍持原武器
- 丢弃的武器落地为可拾取物（不消失）

### 5.3 武器品质视觉区分

| 稀有度 | 边框色 | 武器底色滤镜 | 地面拾取光柱 |
|---|---|---|---|
| Common | 白色 `#FFFFFF` | 无 | 无 |
| Rare | 蓝色 `#4488FF` | 轻蓝光晕 | 蓝色光柱 |
| Epic | 紫色 `#AA44FF` | 紫光脉冲 | 紫色光柱 |
| Legendary | 金色 `#FFD700` | 金光流动 | 金色光柱 + 粒子 |

具体光效规格由 art-vfx 落地，本文档仅定义色值与等级映射关系。

## 六、AI 行为侧需求

### 6.1 Smart AI（8–10 个实例）

Smart AI 拥有武器决策模型（`BotBuildPlanner` 的武器子模块），行为如下：

**武器拾取决策**：

```
拾取优先级 = BaseDamage × rarityWeight × classAffinityScore
```

- `rarityWeight`：Common=1.0 / Rare=1.5 / Epic=2.2 / Legendary=3.0
- `classAffinityScore`：AI 根据当前纹身 build 偏向性打分；右臂蓄力纹身多 → 弓得分高；纹身触发频率型 → 能量拳得分高

**武器切换（按敌人距离）**：

| 当前目标距离 | 偏好武器 Class | 决策阈值 |
|---|---|---|
| < 3m | Melee / Special | 优先切换近战 |
| 3–10m | Ranged | 优先切换远程 |
| > 10m | Ranged（弓） | 切换弓，准备蓄力 |

切换决策每 `0.5s` 评估一次（`BotRethinkInterval = 30 帧`），不实时切换，避免 AI 频繁换武器看起来失控。

**弹药管理**：Smart AI 弹药低于 `MaxAmmo × 0.2` 时主动寻找同类武器补充；若无远程备用，自动切换近战武器而非空弹状态攻击。

### 6.2 Light AI（~40 个实例）

- 出生时分配一把预设武器（不运行期换武器）
- 持近战武器的 Light AI 只在距离 < 4m 时攻击
- 持远程武器的 Light AI 在距离 3–15m 时攻击，近身时仍用远程（不切换）
- 不做弹药管理，弹药耗尽后停止攻击（行为退化，符合 Light AI 设计意图）

## 七、风险与开放问题

### 7.1 传说武器高 BaseDamage 是否导致瞬秒

**问题**：传说武器 BaseDamage 上限 80，t=25 右臂红×直线×化身形态：

```
effectMagnitude ≈ 80 × 0.8 × 1.0 × 1.10 × (1.60 × 1.15 × 1.15)
               ≈ 80 × 0.8 × 1.10 × 2.116 ≈ 149 点/击
```

玩家后期 playerEffHP(25) = 250，一击接近 60% 血量，配合 1 次暴击（×1.5）≈ 223 点，理论两击击倒。

**设计判断**：后期决赛圈允许高爆发，单次暴击不应瞬杀（250 × 1.0 > 149），需两击，符合 12-数值平衡 §一"后期 PVP 击杀时间 ≥ 4 秒"约束（两击需约 0.5s 普攻间隔 + 换位）。目前属于**可接受范围，但需 playtest 实测**。

**缓解**：若实测仍感觉过快，优先降低 `Legendary.BaseDamage` 上限至 65，或降低 `ResonanceMul6`（来自 BalanceConstantsConfig），不修改武器帧表。

### 7.2 弹药稀缺度设计

**推荐（MVP 定案）**：

- **近战武器**：无限弹药（`MaxAmmo = -1`）
- **远程武器**：有限弹药（手枪 18 发，弓 12 箭）
- 弹药补充：击杀怪物/玩家低概率掉落，宝箱有弹药包配置（定义在 08-宝箱 GDD 中）

**不做**：子弹单独捡拾系统（增加 UI 复杂度，MVP 暂不考虑）

**弹药耗尽降级行为**：远程武器弹药耗尽后，武器仍可挥砸使用（BaseDamage × 0.4，无投射物生成），给玩家保底反制手段，避免"完全失去攻击能力"的负体验。

### 7.3 武器升级系统（推荐 MVP 不做）

**为什么不做**：武器升级会引入额外货币消耗路径与 UI，与纹身刻图系统争夺玩家精力，在 MVP 验证核心循环前增加了系统复杂度。

**推迟至 v1.1 的备选方案**：
- 方案 A：材料强化（使用废金属 + 颜料升武器稀有度，共用纹身颜料资源）
- 方案 B：附魔（武器附加一个固定元素属性，与纹身颜色联动加成）

**当前 MVP 定案**：武器只通过替换（找到更高稀有度武器）来升级，不存在强化系统。

### 7.4 冷却武器（推荐 MVP 不做）

部分特殊武器设计为每次使用后有独立冷却（如能量炮）。MVP 期不引入，原因：冷却管理 UI 与技能槽冷却 UI 会混淆，玩家认知负担高。v1.1 可作为特殊系武器扩展。
