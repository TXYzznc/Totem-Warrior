# 背景图错乱问题排查与修复记录

排查日期：2026-06-29
触发：用户怀疑 ShopForm / SelfTattooForm / RunResultForm 三个文件夹的 `_bg.png` 有问题，要求全量核查 `Assets/Resources/Sprite/UI/`。

## 根因

`codex exec` 内部用 PowerShell `Get-ChildItem ... | Sort-Object LastWriteTime -Descending | Select-Object -First 1` 取"最近生成的文件"再复制到目标路径。当同一个 codex 会话或相邻会话里连续生成多张图（先出某个表单的组件合并画布，再出另一个表单的背景）时，这个"取最新 1 张"的逻辑会在时序错位时**抓错文件**——把上一个任务剩下的绿幕组件画布当成"刚生成的背景"复制走。这是 `ui-asset-splitting` 流程里此前没有写明的新坑。

## 修复方式

对 5 个问题 Form 逐一重新调 Codex 生成纯场景背景图（不含任何 UI 面板/图标/文字），**每张生成后先用 `Read` 工具肉眼核验内容**，确认无误后才替换：

| Form | 新背景描述 | 旧文件处理 |
|---|---|---|
| ShopForm | 暗色石质地牢拱廊+紫雾+双火把 | 重命名为 `ShopForm_bg_BROKEN_was_greenscreen.png` 保留备查 |
| RunResultForm | 战后余烬战场，暗红火光，低沉色调 | 重命名为 `RunResultForm_bg_BROKEN_was_greenscreen.png` 保留备查 |
| SelfTattooForm | 暗色战斗竞技场，蓝紫魔法光雾 | 重命名为 `SelfTattooForm_bg_BROKEN_was_greenscreen.png` 保留备查 |
| TattooEnchantForm | 暗色石柱地牢竞技场，紫色神秘光雾 | 重命名为 `TattooEnchantForm_bg_BROKEN_was_panelframe.png` 保留备查 |
| TattooStudioForm | 暗色石柱地牢竞技场，紫粉魔法光雾（右侧） | 全新生成（原本没有） |

全部替换后同步覆盖到 `Assets/Resources/Sprite/UI/<FormName>/`，并用 PIL 校验 size/mode 正常。
