# 游戏 2D 美术通用规范（图片）

> **定位**：规定 2D 游戏美术素材的**类型分类、最大尺寸、格式、文件大小预算**。是美术**生产侧**的前置约束，目标是「生产即合规」，让事后压缩变成例外而非常态。
>
> **范围**：**项目无关的通用规范**，只按视觉用途分类，不绑定任何具体业务术语。各项目可在自己的 `资源配置规范.md` 中定义"放在哪、怎么加载"，本文档只定义"多大、多重、什么格式"——两者正交。

---

## 零、使用场景（关键）

本规范有**两个使用位面**，**前置位面是主战场**：

### 0.1 前置：美术需求 / 提示词阶段（强制 — 主战场）

写美术需求文档 / AI 出图提示词 / 调出图工具参数前 → **必先**按本规范确定：

- 资源的**视觉用途**归到 §二 哪一子类
- 该子类的 **Max 尺寸 / 格式**作为出图工具的目标分辨率参数
- 该子类的**文件大小上限**作为「需要压缩与否」的判定阈值

调用 `ai-art` / `codex-image-gen` / `drawing-prompt-*` 等 SKILL 时，规范都应作为**输入约束**先读再写提示词。按规范生产的图绝大多数本身就达标。

### 0.2 后置：兜底压缩（例外）

只在以下场景才用 `image-compression` SKILL 兜底：

- 外部下载素材 / 历史遗留素材入库前
- 出图工具偶发超规（如某 AI 模型只能产 1024 但目标是 256）
- 老项目对存量素材的批量整改

> **不要**把 `image-compression` 当成"出图后必经流程"。它是兜底的手，不是流水线的环节。生产侧合规了，绝大多数图根本不需要二次压缩。

## 一、通用约束（所有类型适用）

1. **格式优先级**：透明需求 → PNG；无透明大图 → JPEG；动效/序列帧/8bit → WEBP（按需）；TGA 仅用于第三方资源中转，不入正式库
2. **POT（Power of Two）**：移动端纹理压缩（ETC2/ASTC）友好 → 强烈推荐 64 / 128 / 256 / 512 / 1024 / 2048。**桌面端不强制**，但小尺寸图标类必须 POT
3. **mipmap**：UI 类**关闭**；3D 贴图开启
4. **alpha 通道**：能不带就不带（JPEG > PNG-RGB > PNG-RGBA 文件体积排序）
5. **色深**：8bit/channel 即可；除非 HDR / normal map，否则不要 16bit
6. **DPI / @2x**：默认单套素材覆盖；只在高 PPI 屏适配明确立项时再做高分辨率分组
7. **必须压缩**：源图入库前**全部**走一遍 `cli.py --preset <对应类别>`；裸 PNG 直入库视为违规
8. **元数据**：删除 EXIF / 拍摄信息 / 隐私字段，除非业务明确需要

## 二、各类型尺寸 / 格式 / 大小预算

> "Max 尺寸" = 长边或宽边的上限；超过必须缩。"文件大小上限" = 单张文件在磁盘的体积，超过必须二次压。
>
> 移动端独占项目可在此基础上**全局再砍 30-50%**（GPU 纹理内存 + 包体 + 下载时长更敏感）。

### 2.1 UI 元素

界面控件、HUD、菜单组件等纯 UI 用途素材。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 小图标 / 控件图 | **256×256**（POT） | PNG | **≤ 50 KB** | `ui-icon` | 带透明 |
| 大按钮 / Tab | **512×256** | PNG | **≤ 80 KB** | `ui-button` | 状态变体各一张 |
| 面板底板 / 卡片 | **1024×1024** | PNG | **≤ 150 KB** | `ui-panel` | 优先九宫格 sliced |
| 全屏 / 弹窗背景 | **1920×1080** | PNG | **≤ 500 KB** | `ui-bg` | 带透明可保留 |
| 进度条 / 滑块 | **512×64** | PNG | **≤ 30 KB** | `ui-icon` | 九宫格 |

> UI 设计基准分辨率默认 **1920×1080**。若项目锁定其他基准（如 1280×720 竖屏移动），按比例缩放。

### 2.2 角色立绘 / 头像

具人物外观、表情、形象展示用途的素材，与具体角色类型（玩家 / NPC / 怪物 / 主角等）无关。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 全身立绘 | **1024×2048** | PNG | **≤ 500 KB** | `character-portrait` | 不量化保细节 |
| 半身像 / 对话框图 | **768×1024** | PNG | **≤ 300 KB** | `character-portrait` | — |
| 头像 / 圆框 | **512×512**（POT） | PNG | **≤ 100 KB** | `character-avatar` | — |

### 2.3 功能性图标

所有"小尺寸方形、信息密度高、需在 UI 中识别"的图。无论业务上叫物品 / 武器 / 技能 / 状态 / 货币 / 成就 / 配方……视觉规格统一。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 通用图标（小） | **256×256**（POT） | PNG | **≤ 80 KB** | `item-icon` 或 `skill-icon` | 二者参数一致，按业务习惯任选 |
| 大图标 / 详情图 | **512×512**（POT） | PNG | **≤ 150 KB** | `item-icon` | 物品/技能详情面板使用 |

> `item-icon` 与 `skill-icon` 两个 preset 参数完全相同，只为方便 AI 按上下文语义命名时不歧义；不必拘泥某类业务必须用某个 preset。

### 2.4 贴花 / 装饰图样 (Decal)

贴在角色或物体**表面**的图样：可能是涂鸦、印记、自定义图样、贴纸、徽章、装饰花纹等。视觉特点是**细节丰富、不能量化（量化会丢边缘 / 渐变）**。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 完整贴花 | **1024×1024**（POT） | PNG | **≤ 250 KB** | `decal` | 不量化，保细节 |
| 组件 / 局部贴花 | **512×512**（POT） | PNG | **≤ 120 KB** | `decal` | — |

### 2.5 场景 / 背景插画

整屏铺底、不需要透明、信息密度高（细节多色彩多）的大图。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 场景大背景 | **1920×1080** | **JPEG q85** | **≤ 800 KB** | `environment-bg` | 无 alpha 用 JPEG 显著省 |
| 远景 / 天空盒 | **2048×1024** | JPEG q85 | **≤ 800 KB** | `environment-bg` | — |

### 2.6 特效贴图

粒子系统 / 序列帧动画的贴图源。

| 子类 | Max 尺寸 | 格式 | 文件大小上限 | preset | 备注 |
|---|---|---|---|---|---|
| 粒子贴图 | **512×512**（POT） | PNG | **≤ 80 KB** | `effect-texture` | 带透明 |
| 序列帧（每帧） | **256×256**（POT） | PNG | **≤ 40 KB/帧** | `effect-texture` | 优先打包为 Sprite Atlas |

### 2.7 其它

| 子类 | 处理 |
|---|---|
| 截图 / 参考图（非入库） | `screenshot` preset 压到 ≤ 300KB 后再贴对话 |
| 第三方下载素材 | 入库前先按上表归类、压缩、重命名 |
| 3D 贴图（albedo / normal / roughness 等） | 不在本规范范围；走 PBR / texture-art 流程 |

## 三、Definition of Done（资源入库前自检）

任何一张新素材在 commit 前，作者（人或 AI）必须确认：

- [ ] 视觉用途归到 §二 某个子类（与业务命名无关，只看图本身）
- [ ] 尺寸 ≤ 该子类 Max 尺寸
- [ ] 文件大小 ≤ 该子类上限
- [ ] 格式选择正确（带透明用 PNG / 大背景无透明用 JPEG）
- [ ] 已用 `cli.py --preset <类别>` 处理过（或人工确认无需压缩）
- [ ] 文件名遵循各项目规范（PascalCase / snake_case 等，看所在子目录现有约定）
- [ ] 未带 EXIF / 隐私元数据（除非明确需要）

**违规处理**：

- AI agent 触发 `image-compression` skill 时按本规范执行；超标必告警，不可静默通过
- code review / git pre-commit 检测到新增素材文件超规 → 拒绝合入并指向本规范

## 四、压缩工具 preset 速查（兜底用）

`tools/ImageCompression_Tool/cli.py` 内置一组方便的压缩参数预设，名字与本规范的视觉分类对齐。**只在 §0.2 兜底场景使用**；正常情况下规范前置生产，根本走不到这里。

| preset | Max 尺寸 | 格式 | 量化 | 适用的视觉类别 |
|---|---|---|---|---|
| `ui-icon` | 256×256 | PNG | ✓ | UI 小图标 / 进度条 |
| `ui-button` | 512×256 | PNG | ✓ | 大按钮 / Tab |
| `ui-panel` | 1024×1024 | PNG | ✓ | UI 面板 / 卡片底板 |
| `ui-bg` | 1920×1080 | PNG | ✓ | UI 全屏 / 弹窗背景 |
| `character-portrait` | 1024×2048 | PNG | ✗ | 角色全身 / 半身 |
| `character-avatar` | 512×512 | PNG | ✓ | 角色头像 |
| `item-icon` | 256×256 | PNG | ✓ | 功能性图标（小） |
| `skill-icon` | 256×256 | PNG | ✓ | 功能性图标（小，等价于 item-icon） |
| `decal` | 1024×1024 | PNG | ✗ | 贴花 / 装饰图样 |
| `environment-bg` | 1920×1080 | JPEG q85 | — | 场景大背景 / 天空盒 |
| `effect-texture` | 512×512 | PNG | ✓ | 粒子 / 序列帧 |
| `screenshot` | 1920×1080 | JPEG q80 | — | 截图 / 参考图 |

> preset 是「兜底压缩工具的便利预设」，**不是**本规范的实质。规范本身脱离压缩工具独立存在；新增视觉类别先改本文档 §二，preset 表是否同步只看是否要让 image-compression 也覆盖该类。

## 五、待商榷项（v1 → v2 候选）

- [ ] **Sprite Atlas 全面启用**：UI / 图标 / 特效类频繁同屏渲染，启 Atlas 后批次能大幅下降，但要新增运维流程
- [ ] **@2x / @3x 高分辨率分组**：默认单套覆盖；后续若适配高 PPI 平板 / 桌面再补
- [ ] **WEBP 替代 PNG**：WEBP 文件体积通常省 25-50%，但 Unity 需插件支持、编辑器预览不直观
- [ ] **Addressables / 远程 CDN**：超大背景资源是否走远程下载，取决于发行形态
- [ ] **目录总体积上限**：是否对每个素材子目录设总和上限（如背景类总和 ≤ 20MB）
