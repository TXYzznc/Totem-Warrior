
# Drawing Prompt Generator — 总入口

## 职责

分析用户的绘图需求，**判断类型**，读取对应子模块的规范，生成符合当前项目美术风格的 AI 绘图提示词。

> 各子模块的"核心设计理念"在项目美术风格初始化前为空。若子模块参考文件仍处于待初始化状态，生成提示词时应优先采用用户当前请求中给出的风格描述，或先引导用户执行 `SKILL.md` 中的「美术风格初始化流程」。

## 类型判断规则

| 用户描述关键词 | 对应子模块 | 读取文件 |
|--------------|-----------|---------|
| 技能图标、Buff图标、技能、Buff、skill icon | **ICON** | `ICON/SKILL.md` |
| 角色、立绘、人物、召唤师、英雄、NPC、头像、portrait | **CHARACTER** | `CHARACTER/SKILL.md` |
| 场景、背景、地图、环境、战场、landscape | **SCENE** | `SCENE/SKILL.md` |
| UI、界面、按钮、面板、边框、HUD、弹窗 | **UI** | `UI/SKILL.md` |
| 子弹、道具、树木、岩石、特效素材、载具、单位、prop、bullet、item（不属于上述任一类型的通用 2D 素材） | **COMMON** | `COMMON/SKILL.md` |

## 工作流程

1. **读取用户需求** — 提取绘图目标的核心描述
2. **判断类型** — 根据上表匹配对应子模块
3. **读取子模块** — 加载对应 `SKILL.md` 中的生成规范
4. **执行生成** — 按子模块规范进行推理并输出提示词
5. **输出结果** — 结构化提示词 + 简要推理说明

## 输出附加要求（透明背景 PNG）

**ICON、CHARACTER、UI、COMMON** 类型生成的最终提示词，必须在结尾附加以下透明背景要求（各子模块模板已包含，生成时不要省略）：

```
请生成真正透明背景的 PNG 图片。

要求：
1. 背景必须是真实透明通道，也就是 PNG RGBA 的 alpha=0。
2. 不要白底、不要灰底、不要棋盘格、不要"模拟透明背景"。
3. 画面中只能保留主体图形本身，主体外所有区域必须完全透明。
4. 生成后请用程序检查图片是否为 RGBA，并验证四个角落像素 alpha=0。
5. 如果生成结果带有棋盘格，请不要直接交付，请先后处理移除背景并导出真实透明 PNG。

这个提示词能保证AI绘制的图片是真正的PNG格式图片。
```

**SCENE（场景图）不附加此要求** — 场景图是铺满整个画布的背景画面，不存在"主体外透明区域"的概念，强加透明背景要求会与 `Background layers` 结构冲突。

## 模糊情况处理

- 若无法判断类型，直接询问用户
- 若需求跨多个类型（如"技能释放场景图"），优先以主体对象判断（场景→SCENE）
- 同一次请求可同时生成多个类型（如"技能图标+对应场景背景"）
