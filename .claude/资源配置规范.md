# 资源配置规范

本文档规范项目中所有游戏资源（美术、音频、预制体等）的组织方式、命名规范、加载机制。

---

## 📋 核心原则

1. **集中管理**：所有资源存放在 `Assets/Resources/` 下，通过 `ResourceConfig` 配置表统一管理
2. **配置驱动**：代码不硬编码资源路径，全部通过 ResourceConfig ID 加载
3. **类型划分**：按资源类型区分子目录（Sprite、Audio、Prefab、Font、Texture 等）
4. **路径唯一**：同一资源只存一份，不同的美术版本用配置表的 LoadPath 变量管理

---

## 🗂️ 目录结构规范

### 标准结构

```
Assets/Resources/
├── Animation/          ← 动画片段 (.anim)
├── Audio/             ← 音频文件 (.mp3, .wav)
├── DataTable/         ← 配置表数据 (.json)
├── Effect/            ← 特效资源
├── Font/              ← 字体文件 (.ttf, .otf)
├── Material/          ← 材质 (.mat)
├── Model/             ← 3D模型 (.fbx, .obj)
├── Prefab/            ← 预制体 (.prefab)
├── Sprite/            ← 2D精灵 (.png, .jpg)
│   ├── Characters/    ← 角色精灵
│   │   ├── Player/    ← 玩家角色
│   │   └── Enemies/   ← 敌人角色
│   ├── Bullets/       ← 子弹精灵
│   ├── UI/            ← UI元素精灵
│   │   ├── Backgrounds/
│   │   ├── Buttons/
│   │   └── HUD/
│   └── Effects/       ← 特效贴图
└── Texture/           ← 纹理贴图 (3D用)
```

### 目录命名规则

| 规则 | 说明 |
|------|------|
| **大小写** | 子目录用 PascalCase（Characters, Bullets） |
| **单复数** | 优先用复数（Characters, Sprites，除非是单一类型） |
| **避免特殊符号** | 不用空格、特殊字符，使用下划线或驼峰 |
| **层级不超过3级** | Sprite → Characters → Player（可以，3级）<br>Sprite → A → B → C（不行，超过3级） |

## 📝 资源命名规范

### 通用规则

| 类型 | 格式 | 示例 | 说明 |
|------|------|------|------|
| **Sprite** | `[context]_[name].png` | `player_ship.png` | 英文下划线连接 |
| **Prefab** | `[name]Prefab.prefab` | `EnemyPrefab.prefab` | PascalCase，后缀 Prefab |
| **音频** | `[context]_[name].mp3` | `sfx_explosion.mp3` | 小写下划线 |
| **字体** | `[FontName].ttf` | `Arial.ttf` | 保留原字体名 |
| **配置表** | `[TableName]Config.json` | `EnemyConfig.json` | PascalCase + Config 后缀 |

### Sprite 命名示例

```
Characters/
├── Player/
│   └── player_ship.png           ← 玩家飞机
├── Enemies/

│   └── enemy_boss.png            ← Boss敌人

Bullets/
├── player_bullet.png             ← 玩家子弹
└── enemy_bullet.png              ← 敌人子弹

UI/
```

## 🔗 加载机制（ResourceModule）

### 工作流程

```
1. 定义资源信息
   ↓
2. 在 ResourceConfig.json 中添加行
   { "Id": 1001, "Name": "PlayerShip", "Type": "Sprite", "LoadPath": "Characters/Player/player_ship" }
   ↓
3. 代码通过 ID 加载
   var playerShip = resourceModule.Load<Sprite>(1001);  // 自动加载 Assets/Resources/Sprite/Characters/Player/player_ship.png
```

### 规则

| 规则 | 说明 |
|------|------|
| **Type** | 必须是 Resources 下的一级目录名（Sprite, Audio, Prefab, Font 等） |
| **LoadPath** | 相对路径，不含扩展名，从 Type 目录开始<br>例：Characters/Player/player_ship |
| **缓存** | ResourceModule 自动按 ID 缓存，重复加载返回缓存副本 |
| **路径检验** | 资源不存在时返回 null 并打印 Error 日志 |

## 📊 ResourceConfig 配置表

### 表结构

```json
{
  "table": "ResourceConfig",
  "fields": [
    { "name": "Id", "type": "int", "desc": "资源ID，主键，建议用区间划分（1000-1999: Sprite, 2000-2999: Audio 等）" },
    { "name": "Name", "type": "string", "desc": "资源标识名，便于阅读和查找" },
    { "name": "Type", "type": "string", "desc": "资源类型，对应Resources下子文件夹名" },
    { "name": "LoadPath", "type": "string", "desc": "相对Resources/{Type}/的加载路径，不含扩展名" }
  ],
  "rows": [
    { "Id": 1001, "Name": "PlayerShip", "Type": "Sprite", "LoadPath": "Characters/Player/player_ship" },
    { "Id": 1002, "Name": "EnemyScout", "Type": "Sprite", "LoadPath": "Characters/Enemies/enemy_scout" }
  ]
}
```

### ID 分配规则

建议按资源类型用区间划分：

| 范围 | 类型 | 示例 |
|------|------|------|
| 1000-1999 | Sprite | 1001: PlayerShip, 1002: EnemyScout |
| 2000-2999 | Audio | 2001: SFX_Explosion, 2002: BGM_Menu |
| 3000-3999 | Prefab | 3001: BulletPrefab |
| 4000-4999 | Font | 4001: Arial |
| 5000-5999 | 其他 | 保留扩展 |

## 🎨 美术资源特殊规范

### Sprite 导入设置

所有 Sprite 统一配置：

```
Texture Type: Sprite (2D and UI)
Sprite Mode: Single
Pixels Per Unit (PPU): 100
Filter Mode: Point (保留像素风格)或Bilinear (平滑)
Compression: RGBA Compressed (如需透明) 或 RGB Compressed
```

### 透明背景要求

- **格式**：PNG with alpha channel (RGBA)
- **背景**：真实透明 (alpha=0)，不是白色或灰色
- **用途**：游戏中的角色、敌人、子弹等UI元素需要透明背景

### 尺寸规范

每个项目定义标准尺寸，例如 STG 游戏：

| 元素 | 尺寸 | PPU | 游戏单位 |
|------|------|-----|--------|
| 玩家飞机 | 120×120 | 100 | 1.2×1.2 |
| 敌人（小） | 80×80 | 100 | 0.8×0.8 |
| 子弹 | 20×20 | 100 | 0.2×0.2 |
| 按钮 | 200×60 | UI坐标 | - |

## 🔄 常见场景

### 场景1：新增一个 Sprite

```
1. 美术交付 player_ship.png 到 Assets/Resources/Sprite/Characters/Player/
2. 在 ResourceConfig.json 添加：
   { "Id": 1001, "Name": "PlayerShip", "Type": "Sprite", "LoadPath": "Characters/Player/player_ship" }
3. 运行 DataTableGenerator 更新 ResourceConfig.cs
4. 代码中使用：
   var ship = resourceModule.Load<Sprite>(1001);
```

### 场景2：音频资源

```
位置：Assets/Resources/Audio/sfx/explosion.mp3
配置：{ "Id": 2001, "Name": "SFXExplosion", "Type": "Audio", "LoadPath": "sfx/explosion" }
代码：var clip = resourceModule.Load<AudioClip>(2001);
```

### 场景3：批量更新 LoadPath

如果美术文件重新组织，直接修改 ResourceConfig.json 的 LoadPath 字段，不需要修改代码。

## ⚠️ 常见错误

| 错误 | 后果 | 解决方案 |
|------|------|--------|
| LoadPath 包含扩展名 | 加载失败 | 移除 `.png`、`.mp3` 等 |
| Type 大小写不对 | 找不到目录 | 确保与 Resources 下目录名完全一致 |
| LoadPath 以 `/` 开头 | 路径错误 | 不加前导斜杠 |
| 资源在 Assets/Art/ | 加载失败 | 移动到 Assets/Resources/Sprite/ |
| 代码硬编码 "Assets/Resources/Sprite/..." | 路径耦合 | 用 ResourceConfig ID 代替 |
