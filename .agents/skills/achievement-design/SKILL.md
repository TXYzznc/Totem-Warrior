# Achievement / Trophy Design

## 何时使用
- 设计游戏成就 / 奖杯 / 徽章列表时
- 上架 Steam / PSN / Xbox 前对齐平台规范
- 评审成就分布是否过于偏向「通关型」而缺乏「挑战型」
- 设计 missable / hidden 成就的提示策略
- 跨平台进度同步（云存档、Steam → Epic 迁移）

## 核心原则
1. **成就是第二条游戏循环**：通关后让玩家有再开一局的理由。
2. **每个成就都讲一个故事**：好的成就名 + 描述 = 一句游戏文学（例：Dark Souls「Bond of a Pale Shade」）。
3. **稀有度不是平均分布**：让 80% 玩家拿到通关，让 1% 玩家拿到白金。世界纪录榜要有人爬。
4. **不惩罚错过**：missable 成就必须给玩家**事先**警告，或允许 NG+ 补完。Witcher 3、Persona 5 都遵循这点。
5. **成就映射玩法多样性**：通关、探索、挑战、收集、社交、彩蛋——六大类至少覆盖四类。

## 关键模式

### 模式 A：成就分类六维
| 分类 | 含义 | 占比建议 | 案例 |
|---|---|---|---|
| Completion（通关） | 推进主线、章节、结局 | 25-35% | 「击败最终 Boss」 |
| Skill（技巧） | 无伤、限时、特殊条件 | 10-20% | Dark Souls「不死人」 |
| Exploration（探索） | 地图、收集、隐藏区域 | 15-25% | Hollow Knight「绘图师」 |
| Collection（收集） | 全武器、全道具、全图鉴 | 10-15% | Pokémon「Pokédex 完成」 |
| Hidden（隐藏） | 描述模糊或彩蛋 | 10-15% | Hades「Slashed Benefits」 |
| Missable（限时） | 必须特定路线/选项 | <10% | Persona 5「全社群」 |

### 模式 B：稀有度 / 点数分级（以 PSN 为标准）
- **Bronze（铜，常见）**：5-10 个，主线推进、入门挑战。预期解锁率 70-95%。
- **Silver（银，进阶）**：8-15 个，支线完成、中等挑战。预期 30-60%。
- **Gold（金，专精）**：2-5 个，重大里程碑、困难成就。预期 10-25%。
- **Platinum（白金）**：1 个，全成就解锁。预期 1-5%（白金率是工作室口碑指标）。

Steam 没有铂金，但显示**全球解锁百分比**——稀有成就（<5%）会带「Ultra Rare」标签，玩家追这个。

### 模式 C：触发设计——明示 vs 暗示
**明示型**（描述完全公开）：
- 用于通关、收集、明显挑战
- 玩家知道目标，努力达成是乐趣
- 例：「击败 100 个敌人」

**暗示型 / 隐藏**（描述被隐藏，只显示「???」）：
- 用于剧透节点、彩蛋、惊喜
- 解锁瞬间是「Aha!」时刻
- 例：Hades 大量剧情成就描述被遮蔽
- **平台规范**：Steam 允许 hidden，PSN/Xbox 允许 secret trophy

**反例**：把全收集做成 hidden——玩家不知道在找什么，只会卡死。收集类必须明示。

### 模式 D：Missable 警告策略
Missable 成就是高风险——玩家通关一次发现错过，会有强烈挫败。解决方案：
1. **NG+ 补完**：所有 missable 在 NG+ 中可继续追（Persona 5）
2. **章节归档**：每章结束前提示「本章可解锁成就 3/5」（Life is Strange）
3. **章节回放**：通关后可选择回到任意章节（Telltale games）
4. **完全避免 missable**：保持开放世界永久可访问（Skyrim 几乎无 missable）

发售前需明确告知玩家：成就指南页 / Steam 公告 / 游戏内 tooltip。

### 模式 E：平台规范差异
| 平台 | 数量限制 | 点数体系 | 限制 |
|---|---|---|---|
| Steam | 无硬限（建议 < 100） | 无官方点数，靠全球稀有度 | 必须可解锁，不可只用于装饰 |
| PSN | 必须包含 1 个白金 | Bronze/Silver/Gold/Platinum | 总点数固定（白金=180/金=90/银=30/铜=15） |
| Xbox | 总 Gamerscore 1000 起步 | 单成就 5-200G | DLC 可额外添加 |
| Nintendo Switch | **无官方成就系统** | - | 需游戏内自建 |
| Epic / GOG | 各有 SDK | 类似 Steam | 兼容性需自测 |

**坑**：PSN 白金不能只是「解锁所有其他成就」字面意思——必须显式实现 unlock logic。

### 模式 F：命名风格
好的成就名 = **隐喻 + 双关 + 游戏内黑话**：
- Dark Souls「Bond of a Pale Shade」——只对玩过 DLC 的玩家有意义
- Hollow Knight「Beast」——一字双关（兽性 / 主角形态）
- Hades「Is There No Escape?」——重复死亡的玩家自嘲
- Celeste「Farewell」——告别既指章节也指开发组与玩家

**避免**：直白叙述（「击败 Boss A」）、英文直译中文（「成就解锁者」）。

### 模式 G：进度同步与持久化
- **存档隔离**：成就应跨存档（账号级，非档位级）——避免「换档丢成就」
- **云同步**：Steam Cloud / PSN Cloud 自动同步，但需在 SDK 调用前调用 `unlock` API
- **断网容错**：本地预解锁 + 联网后回传，防止解锁瞬间断网导致丢失
- **重复解锁**：API 调用 `unlock(id)` 在已解锁状态下应静默成功，不要崩溃

## 常见坑
- **坑：白金率 0%**。原因：某个成就 bug 或条件过苛。修法：发售前内部至少 3 人完成白金路径验证。
- **坑：missable 玩家炸锅**。原因：没有任何警告。修法：tooltip + 章节归档 + NG+ 补完，三选一最低。
- **坑：成就描述剧透**。原因：第十章成就显示在第一章成就墙上。修法：分组 / 隐藏 / 描述模糊。
- **坑：刷成就 bug**。原因：成就触发依赖客户端状态。修法：服务端验证 + 反作弊检测（多人游戏尤其重要）。
- **坑：DLC 成就拉低完成率**。原因：把 DLC 成就计入基础完成率，玩家看到 60% 心态崩。修法：分基础 / DLC 两个统计。
- **坑：「玩满 100 小时」型成就**。原因：纯耗时不创造体验。修法：替换为「通关 5 次不同结局」「使用全部武器至少一次」等行为型。
- **坑：成就名本地化丢双关**。原因：直译。修法：本地化时允许重写（不是翻译），保留文学性。

## 输出形式

### 1. 成就列表表（Excel / Notion）
| ID | 名称 | 描述 | 隐藏 | 分类 | 稀有度 | 平台点数 | 触发条件 | DoD |
|---|---|---|---|---|---|---|---|---|
| ACH_001 | First Blood | 击败第一个敌人 | 否 | Completion | Bronze | 15G | enemies_killed >= 1 | ☐ 文案 ☐ 触发 ☐ 本地化 |

### 2. 稀有度分布饼图
检查是否符合 70/20/10 法则（70% 通关 / 20% 进阶 / 10% 极限）。

### 3. Missable 警告清单
列出所有 missable 成就 + 警告时机 + NG+ 是否可补。

### 4. 平台兼容性矩阵
每个成就在 Steam/PSN/Xbox 上的对应 ID、点数、稀有度映射，发售前由 QA 逐项过。
