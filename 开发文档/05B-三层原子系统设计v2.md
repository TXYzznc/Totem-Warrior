## 一、合成规则（唯一一条）

```
事件 E 发生时，对每个 part.triggerEvent == E 的槽位执行：

  1. part.PrepareContext(ctx)                          // 重塑上下文：目标/路径/反弹源
  2. magnitude = computeMagnitude(slot, ctx, synergy)  // 基础数值（缩放×颜色×图案×联动）
  3. magnitude = color.element.ModifyMagnitude(ctx, magnitude)  // 颜色调整倍率（白色 +30%）
  4. handled = part.InterceptApply(ctx, player, shape, element, magnitude)
       如果 handled = true（部位拦截 → 打包为 PendingTrigger），本次结束
  5. 否则 pattern.shape.Apply(ctx, color.element, magnitude)
  6. color.element.AffectSelf(player, ctx, magnitude)  // 颜色对自身效果（金回血/白叠专注）
  7. part.AffectSelf(player, ctx)                      // 部位对自身效果（左腿短无敌）

// 事件结束后处理被打包的 PendingTrigger
for pt in player.pendingTriggers where pt.consumeOnEvent == E:
  pt.shape.Apply(ctx, pt.element, pt.magnitude)
  remove pt

// 装备/卸下纹身时自动重算被动属性
player.derivedStats = baseStats + sum(slot.passive() for slot in equipped)
```

## 二、部位层 6 原子（复杂度集中在此）

每个部位 SO 提供 **5 个 hook**，决定何时触发、目标如何重塑、是否延迟、自身效果与被动属性。

| ID | 部位 | 触发事件 | 缩放属性 | 上下文重塑 | 拦截行为 | 被动维度 |
|---|---|---|---|---|---|---|
| **Head** | 脑袋 | OnCrit | 暴击伤害 | — 默认 | — | 暴击率 / 元素暴击伤害 |
| **Torso** | 躯干 | OnDamaged | 生命上限 | `ctx.PrimaryTarget = ctx.LastAttacker` | — | 元素抗性 / 减伤 / 最大生命 |
| **LeftArm** | 左臂 | OnSkillCast | 技能强度 | — 默认 | — | 元素技能强度 / 冷却 |
| **RightArm** | 右臂 | OnAttack | 武器伤害 | — 默认 | — | 元素武器附魔 / 攻速 |
| **LeftLeg** | 左腿 | OnDodgePressed | 闪避无敌帧 | — | **打包成 PendingTrigger，consumeOn = OnAttack** | 闪避帧 / 闪避 CD |
| **RightLeg** | 右腿 | OnMoveTick（移动每 0.5s） | 移动速度 | `ctx.PrimaryTarget = pickForwardEnemy()` | — | 移速 / 位移距离 |

### 关键设计：上下文重塑的语义

| ctx 字段 | 含义 |
|---|---|
| `PrimaryTarget` | 本次效果命中的"主要目标"（图案侧用） |
| `NearbyTargets` | "次要目标"列表（图案侧 AOE/链 用） |
| `LastAttacker` | 玩家最近一次被攻击的来源 |
| `MovementForward` | 玩家当前移动方向，用于挑前方敌人 |

`Head/LeftArm/RightArm` 不重塑 → 默认 `PrimaryTarget = 事件自带目标`：
- OnCrit / OnAttack：`PrimaryTarget = 被打的敌人`
- OnSkillCast：`PrimaryTarget = 技能命中目标`

`Torso` 重塑 → 把攻击者拉过来当目标，实现**反弹**语义。
`RightLeg` 重塑 → 挑朝向最近的敌人，实现**移动放射**语义。

### 关键设计：拦截 + PendingTrigger 的语义

`LeftLeg` 的 `InterceptApply` 返回 true：
```
// 闪避按下时，不立即对外造成伤害
// 把"shape + element + magnitude"打包成"延迟触发"挂到玩家自身
player.pendingTriggers.add(new PendingTrigger {
    consumeOnEvent = OnAttack,  // 下次普攻消耗
    shape          = currentShape,
    element        = currentElement,
    magnitude      = currentMagnitude,
});
return true;  // 跳过 shape.Apply
```

这让"闪避后下次攻击附加 X 效果"这种**延迟激活**机制完全装入原子系统。

## 三、颜色层 7 原子（保持简洁）

每个颜色 SO 持有一个 `ElementBehavior`，提供 **4 个 hook**：

| ID | 颜色 | 元素 | `ApplyElementEffect`（命中目标的副作用） | `ModifyMagnitude`（修改倍率） | `AffectSelf`（自身效果） | 抗性维度 |
|---|---|---|---|---|---|---|
| **Red** | 红 | 火焰 | 燃烧 DOT | 1.0× | — | 火抗 |
| **Yellow** | 黄 | 雷电 | 麻痹 | 1.0× | — | 雷抗 |
| **Green** | 绿 | 自然/毒 | 中毒 DOT | 1.0× | — | 毒抗 |
| **Blue** | 蓝 | 冰霜 | 冰缓 / 累积冰冻 | 1.0× | — | 冰抗 |
| **Purple** | 紫 | 异变 | 随机状态（眩晕/沉默/虚弱）| 0.9×~1.3× 随机 | 自身轻微随机 buff | 紫抗 |
| **Gold** | 金 | 神圣 | 必中 + 标记 | 1.0×（标记后再命中 ×1.5） | **治疗自身 = 伤害量 × 15%** | 神抗 |
| **White** | 白 | 纯能 / 光 | 标记目标"纯能印记"（暴伤 +30%） | **×1.20** | **叠"专注"，每层后续伤害 +1%** | 纯抗 |

**说明**：白色不再是"空元素"，而是**纯能/光**这个独立元素，副作用为**纯能印记**（让被命中目标在后续承伤时再增 30% 暴伤），并且 **白色自带 +20% 倍率 + 自身专注叠层**——这就把文档里"白色=通用百分比增益"和"白色×螺旋=叠专注"这两条具象出来。

## 四、图案层 8 原子（保持最简单）

每个图案 SO 持有一个 `EffectShape`，只暴露一个 `Apply` 方法。**形状只关心"命中谁、几次、用什么倍率"**，不 know 部位也不 know 颜色具体行为。

| ID | 图案 | 命中模式 | 倾向 |
|---|---|---|---|
| **Line** | 直线 | 1 个 PrimaryTarget × 1 次 | 单点重击 |
| **Ring** | 圆环 | PrimaryTarget + 附近最多 4 个，每个 60% magnitude | AOE |
| **Spiral** | 螺旋 | 叠层印记，第 K=5 次命中时引爆 4× magnitude | 叠层引爆 |
| **Zigzag** | 锯齿 | 把 magnitude 拆为 N=4 段，对 PrimaryTarget 各命中 1/N | 多段拆击 |
| **Bolt** | 闪电 | PrimaryTarget → 链跳最多 3 次，每跳衰减 0.7× | 连锁 |
| **Star** | 星形 | 概率 P=0.6 命中（命中时 ×2 magnitude，未中 0） | 概率倍发 |
| **Stream** | 流线 | PrimaryTarget + Nearby 持续 3 tick，每 tick 40% magnitude | 持续区域 |
| **Beast** | 兽形 | PrimaryTarget × 2.5 magnitude + 玩家"召唤" buff | 召唤 |

## 五、336 组合的语义生成规则（自动展开）

任一组合 `(part, color, pattern)` 的最终行为按合成规则自动算出：

```
[何时]   = part.triggerEvent
[目标]   = part.PrepareContext(ctx) 后的 ctx.PrimaryTarget / NearbyTargets
[拦截]   = part.InterceptApply 决定是即时命中还是打包延迟
[基数]   = player.stat(part.scaleStat) × part.ScaleFactor × color.BaseMul × pattern.Mul × synergy
[倍率]   = color.element.ModifyMagnitude（白色 ×1.20）
[命中]   = pattern.shape.Apply（直线打 1 个 / 圆环打 N 个 / 闪电链跳）
[副作用] = color.element.ApplyElementEffect（红=燃烧 / 黄=麻痹 / ...）
[自身效果] = color.element.AffectSelf（金=回血 / 白=叠专注）+ part.AffectSelf（左腿=短无敌）
```

→ **每个组合都是"读这些原子的属性 + 调它们的 hook"自动展开**，没有任何一条需要单独写代码。

## 九、被动属性 modifier

每个槽位的"被动贡献"按以下规则生成（**装备时立即生效，与触发无关**）：

```
slot.passive(playerStats):
    dimension = part.passiveDimension       // 由部位决定（脑袋→暴击；躯干→抗性...）
    element   = color.element               // 由颜色决定（红→火；蓝→冰...）
    delta     = pattern.passiveStrength     // 由图案决定（直线=稳定+X；螺旋=叠层；圆环=高基础值）

    apply delta to (dimension, element) on playerStats
```

例：
- **躯干×红×直线** = 火抗 +25%
- **躯干×红×圆环** = 火抗 +35%（圆环=广泛防护，基础值更高）
- **躯干×蓝×螺旋** = 装满时上限 +50% 冰抗，但需要叠层（受到 1 次冰伤叠 1 层，每层 +10% 冰抗）

## 十二、可能踩雷的接口耦合点

1. **OnHitExtra(ctx, shape, target)** — 颜色 hook 收到 shape 参数（白色×闪电=额外普攻 需要 know shape）。这是有意的耦合：让"特定颜色+图案"产生质变效果时不必再单独类。
2. **PendingTrigger 持有 shape 引用** — 跨事件时 shape SO 必须稳定（不能在游戏过程中被替换）。Solo 开发场景下不是问题。
3. **被动 modifier 的"叠层 buff"语义** — 需要装备时立即开始监听（如蓝×螺旋：受冰伤时叠层）。Phase 1 先实现静态被动（"装备就生效"），叠层动态被动留到 Phase 2。

5. **Phase 1 先实现静态被动**；动态叠层被动（蓝×螺旋）留到 Phase 2
