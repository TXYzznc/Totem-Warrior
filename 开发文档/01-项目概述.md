# 01 项目概述

## 一、核心概念

一句话定义：**以"涂鸦/纹身"为构筑机制的俯视角 2.5D 肉鸽大逃杀联机游戏**。

### 一段话延伸

玩家进入一个由多种末日成因（AI 叛乱、外星入侵、病毒变异）共同造就的废墟世界，扮演实验体角色，与 50 名玩家（含人机 AI）在随机生成的大地图上展开为时 15-25 分钟的生存对抗。玩家通过探索地图获取颜料、配方和图案，在身体 6 个部位刻下不同颜色、不同图案的纹身组合，构建出独一无二的能力 build 与视觉外观。每局结束身上纹身重置，只有解锁的新角色保留——局外成长不影响局内强度，仅扩展可玩内容。

## 二、设计基本盘

| 维度 | 决策 |
|------|------|
| 类型 | Roguelike + Battle Royale |
| 视角 | 俯视角 2.5D |
| 引擎 | Unity |
| 首发平台 | PC（Windows） |
| 兼顾平台 | 移动端（性能预算需提前考量） |
| 剧情强度 | 弱剧情（重氛围 + 系统驱动） |
| 单局时长 | 15-25 分钟 |

| 队伍模式 | Solo 主推（支持邀请好友共战） |
| 开发规模 | Solo / 小团队 |

## 三、核心差异化（USP）

### 1. 纹身构筑系统
- 取代传统装备栏，**身体本身就是 build 展示面**
- 玩家可远观对手的纹身直接判断流派与强度
- 三层正交结构（6×7×8）：自由度与可实现性兼得
- 自由装饰部分独立于机制，让美感成为独立追求

### 2. 死亡宝箱 = 联机肉鸽核心
- 死亡玩家的所有物资在原地生成宝箱
- 制造天然的 PVP 热点与"猎杀链"动机
- 让"杀死强者"成为最高效的资源获取方式

### 3. 多元末日 = 系统差异化
- 同一游戏内同时容纳 AI / 外星 / 病毒三大主题
- 每局地图主题可随机切换 → 显著提升重玩性
- 颜色（元素）与末日成因绑定，让世界观服务于机制

### 4. 局外纯扩展、局内纯肉鸽
- 局外成长不影响数值平衡
- 每局开始公平起点，强度全由局内构筑决定
- 杜绝 P2W 与新老玩家壁垒

## 四、参考游戏基因

| 参考 | 借鉴的部分 |
|------|---------|
| Nuclear Throne | 俯视角射击 / 局内武器肉鸽 / 节奏感 |
| Vampire Survivors | 简洁结构下的深度构筑 / 视觉爆发感 |
| Hades | 流畅动作手感 / 多技能组合 / 操作精度 |
| Dead Cells | 武器多样性 / 闪避帧设计 / 节奏控制 |
| PUBG / Apex Legends | 大逃杀框架 / 缩圈节奏 / 局内资源经济 |

| Hunt: Showdown | 玩家与环境同等危险 / 紧张氛围塑造 |

## 五、目标玩家画像

### 核心受众
- 喜欢肉鸽构筑爽快感的玩家（Nuclear Throne / Hades 用户）
- 喜欢大逃杀紧张节奏但对传统 BR 操作门槛望而却步的玩家
- 喜欢角色定制 / 创造表达的玩家（涂鸦、外观深度自定义）

### 二级受众
- 短局长偏好玩家（15-25 分钟一局比传统 BR 的 30-40 分钟更友好）
- 联机社交玩家（朋友约局，无需固定身份）

## 六、商业模式定位（初步设想）

| 项目 | 倾向 |
|------|------|
| 售卖模型 | 买断制 + 季节性内容更新 |
| 内购 | 仅外观（皮肤、画笔、墨色），**严禁影响平衡** |
| DLC | 新角色 / 新末日主题 / 新图案 / 新地图模板 |
| 反 P2W | 严格遵守"局外不影响局内强度"原则 |

> 该项暂为初步定位，可在原型验证后调整。

## 七、最小可玩版本（MVP）边界

为确保 Solo 开发可控，MVP 阶段建议聚焦：

| 范围 | MVP 含 | MVP 不含 |
|------|--------|---------|
| 玩法 | 单机 + 2-4 人联机 PVE 副本 | 50 人 BR |
| 纹身 | 6 部位 × 1-2 颜色 × 3-4 图案 | 完整 6×7×8 |
| 地图 | 1-2 个主题模板的随机生成 | 全部末日主题 |
| 角色 | 1 个初始角色 | 角色解锁系统 |
| 联机 | Mirror / FishNet 小规模联机 | 大规模专用服务器 |

MVP 完成后再分阶段扩展到完整内容，详见 [`07-技术架构.md`](07-技术架构.md)。

## 八、项目风险概览（高层）

| 风险 | 等级 | 应对方向 |
|------|------|---------|
| 50 人联机对 Solo 开发的技术压力 | 🔴 高 | 分阶段：先 4 人 PVE，再扩展 |
| PVE+PVP 平衡难度 | 🟡 中 | MVP 先验证战斗手感，再调资源经济 |
| 纹身系统实现复杂度 | 🟢 低 | 数据驱动架构已设计，可控 |
| 美术工作量 | 🟡 中 | 图案共享 SVG + 颜色重染，复用一套形状 |
| 玩家学习曲线（系统复杂） | 🟡 中 | 引导循序渐进，先白色 + 基础图案 |

详细风险评估见各专题文档。
