# 模块系统详细设计

---

## 一、IGameModule

```csharp
public interface IGameModule
{
    int ModuleCategory => 4;
    Type[] Dependencies => Type.EmptyTypes;
    UniTask InitializeAsync(CancellationToken ct = default);
    UniTask ShutdownAsync(CancellationToken ct = default);
}
```

`ModuleCategory` 只表示启动优先级：

| Category | 含义 |
|---|---|
| 0 | 启动基础层 |
| 1 | 运行服务层 |
| 2 | 应用协调层 |
| 3 | 项目扩展层 |
| 4 | 后台辅助层 |

---

## 二、依赖规则

- `Dependencies` 只放具体模块类型
- 不支持接口依赖
- `ModuleRunner` 在启动前校验缺失依赖和循环依赖
- 如果模块需要另一个模块先完成，必须显式声明依赖

---

## 三、执行模型

`ModuleRunner` 使用 WhenAny + 持续扫描：

1. 找出依赖已满足的模块
2. 按 `ModuleCategory * 100 - OutDegree` 排序
3. 在并发上限内启动就绪模块
4. 任意模块完成后立刻重新扫描
5. 全部完成后启动结束

---

## 四、关闭顺序

按初始化完成顺序的逆序关闭。

原因：依赖方关闭时，被依赖方通常仍需要可用。

---

## 五、GetModule

`GetModule<T>()` 只用于读取已初始化模块的引用。

约束：

- 只能获取 `Initialized` 状态的模块
- 不应用它绕过事件系统调用项目行为
- 高频只读查询可缓存模块引用

---

## 六、框架内置通用模块

| 模块 | 职责 |
|---|---|
| `SceneModule` | Unity 场景加载/卸载 |
| `FlowModule` | 通用流程注册/切换 |

项目模块不属于框架，应由具体项目自行定义。
