# 事件系统详细设计

---

## 一、核心定位

`EventBus` 是模块间通信工具，负责：

- Fire-and-Forget 通知
- 等待全部订阅者完成
- 请求-响应
- 手动订阅
- 诊断报告

事件类型由框架或项目自行定义。框架事件不得携带项目语义。

---

## 二、通信模式

| 模式 | API | 行为 |
|---|---|---|
| Fire-and-Forget | `Publish<T>(T evt)` | 发出后立即返回 |
| 等待全部 | `PublishAndWaitAsync<T>(T evt)` | 等所有订阅者处理完 |
| 请求-响应 | `RequestAsync<TReq, TRep>(TReq req, TRep defaultReply)` | 取第一个匹配响应者 |
| 手动订阅 | `Subscribe<T>()` | 返回 `IDisposable` |

---

## 三、属性驱动订阅

模块可用 `[EventHandler]` 标记方法，`ModuleRunner` 在模块初始化完成后自动注册，在模块关闭后自动注销。

```csharp
[EventHandler]
void OnSomethingHappened(SomeEvent evt) { }

[EventHandler]
UniTask OnSomethingAsync(SomeEvent evt) { }
```

签名要求：

- 参数有且仅有一个
- 参数必须是引用类型
- 返回类型只能是 `void` 或 `UniTask`

---

## 四、请求处理器

模块可用 `[RequestHandler]` 标记请求处理方法。

```csharp
[RequestHandler]
SomeReply OnQuery(SomeRequest req) { }

[RequestHandler]
UniTask<SomeReply> OnQueryAsync(SomeRequest req) { }
```

---

## 五、诊断报告

`GetDiagnosticReport()` 输出：

- 注册事件类型数
- 注册请求类型数
- 活跃订阅总数
- 手动订阅数
- 最近错误
- 当前订阅明细
- 预扫描但未就绪的订阅者

---

## 六、框架事件边界

框架事件只描述机制事实，例如：

- `SceneLoadStartedEvent`
- `SceneLoadProgressEvent`
- `FlowChangingEvent`
- `StateChangedEvent<TStateId>`

项目事件应放在项目代码目录，不放进框架目录。
