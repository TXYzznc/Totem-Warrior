# AI 友好型日志规范

> 核心原则：允许运行中出问题，但出问题必须可追溯、可 grep、可定位。

---

## 一、统一格式

```text
[模块名|级别] Key=Value Key=Value Location=文件:行号
```

示例：

```text
[SceneModule|INFO] Action=LoadSceneStarted Scene=Bootstrap Mode=Single Location=SceneModule.cs:54
[FlowModule|ERROR] Action=ChangeFlowFailed From=bootstrap To=main Exception=InvalidOperationException Msg="..." Location=FlowModule.cs:111
[StateMachine|INFO] Machine=AppState Action=ChangeState From=<none> To=Ready Location=AsyncStateMachine.cs:61
```

---

## 二、日志级别

| 级别 | 标识 | 何时使用 |
|---|---|---|
| ERROR | `[ERROR]` | 异常、失败、无法继续 |
| WARN | `[WARN]` | 可恢复但值得注意的问题 |
| INFO | `[INFO]` | 生命周期、流程、状态、场景的关键节点 |
| DEBUG | `[DEBUG]` | 默认关闭的详细追踪 |

---

## 三、模块日志重点

| 模块 | 必记字段 |
|---|---|
| `ModuleRunner` | `Module`、`Status`、`Exception` |
| `EventBus` | `Event`、`Handler`、`Exception` |
| `SceneModule` | `Action`、`Scene`、`Mode`、`Progress`、`Exception` |
| `FlowModule` | `Action`、`From`、`To`、`Exception` |
| `StateMachine` | `Machine`、`Action`、`From`、`To`、`Current` |
| `FlowPipeline` | `Pipeline`、`Step`、`Index`、`Action`、`Exception` |

---

## 四、grep 速查

```bash
grep "\[.*|ERROR\]"
grep "\[SceneModule|"
grep "Action=ChangeFlowFailed"
grep "Machine=AppState"
grep "Pipeline=.* Step="
```

---

## 五、边界要求

框架日志只能记录机制层事实，不记录项目语义。

框架日志可以写：

- 加载哪个场景名
- 切换哪个 FlowId
- 哪个状态机从哪个状态 ID 切到哪个状态 ID
- 哪个模块初始化失败

框架日志不应写：

- 具体业务状态含义
- 具体项目场景类型
- 具体项目对象、规则或进度
